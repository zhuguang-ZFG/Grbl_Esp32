# 自动换纸系统说明 (修正版)

## 系统概述

本写字机集成了完整的自动换纸系统，通过74HC595D移位寄存器控制多个步进电机，实现无人值守的连续书写操作。

## 重要修正：M0换纸逻辑

### 修正前的问题
- M0触发时立即开始换纸，"卡"在M0状态
- 上位机等待换纸完成才能发送下一个命令
- 响应性差，流程不清晰

### 修正后的流程
1. **M0指令**: 回到原点，立即发送"ok"
2. **内容检测**: 后台检查M0后是否有后续指令  
3. **等待触发**: 等待下一个G-code命令到达
4. **换纸执行**: 如果有待处理换纸，在新命令执行前完成换纸

## 换纸工作流程 (修正版)

### 自动换纸 (M0指令触发)
1. **M0指令处理**: 回到原点，立即发送"ok"确认M0完成
2. **内容检测**: 2秒内检查M0后是否有后续G-code指令
3. **状态等待**: 系统等待下一个G-code命令到达
4. **换纸触发**: 收到下一个命令时，如果有待处理换纸则启动
5. **出纸阶段**: 面板电机送出写完纸张 (100步)
6. **进纸阶段**: 进纸器送入新纸，监测传感器
7. **传感器触发**: 纸张接触传感器，启动夹纸
8. **夹纸动作**: 压纸电机+面板电机同时启动 (20步)
9. **释放检测**: 纸张离开传感器，所有电机停止
10. **精确定位**: 延时1秒，面板电机前进32步后退2步
11. **完成回复**: 换纸完成，继续执行触发命令

### 工作流程示例
```gcode
G0 X10 Y10 Z20     ; 移动到安全位置
M0                  ; 回原点，发送ok，等待下一个命令
                    ; 上位机收到ok，发送下一个命令
G1 X20 Y20 Z0      ; 触发待处理的换纸，换纸完成后执行
```

### 状态时序图
```
上位机            ESP32                换纸系统
   |                |                      |
 M0  ───────────>│ 检测M0              │
   |                │ 回原点               │
   |<──────── ok ──│ 发送ok              │
   |                │ 检测内容(2秒)       │
 下一个命令 ───────>│ 触发换纸            ───> 启动换纸
   |                │ 等待换纸完成          │
   |                │ 继续执行命令          │
   |                │                      │
```

## 使用方法 (修正版)

### 1. G-code编程
```gcode
G0 X10 Y10 Z20     ; 移动到安全位置
M0                  ; 回原点，发送ok，等待下一个命令
                    ; 上位机收到ok后发送下一个命令
G1 X20 Y20 Z0      ; 触发待处理的换纸，换纸完成后执行
```

### 2. 状态监控
系统会实时输出状态信息：
```
[MSG: Smart M0 detected, checking for content...]
[MSG: Content detected after M0, paper change will trigger on next command]
[MSG: M0 processing completed, OK sent]
[MSG: Paper change pending for next command]
[MSG: Next command detected, starting pending paper change]
[MSG: Paper change state: EJECTING]
[MSG: Paper change completed]
```

### 3. 关键优势
- **响应性**: M0立即返回ok，不阻塞上位机
- **清晰性**: 明确的M0完成和换纸开始分离
- **可靠性**: 换纸在下一个命令前完成，确保准备就绪
- **兼容性**: 与现有G-code发送器兼容良好

## 技术实现细节

### 状态管理
- `m0_detected`: M0命令已检测
- `m0_completed`: M0处理完成，ok已发送
- `pending_paper_change`: 换纸等待下一个命令
- `paper_change_active`: 换纸正在执行

### 关键函数
- `smart_m0_trigger()`: M0检测触发
- `smart_m0_on_next_command()`: 下一个命令触发换纸
- `m0_complete()`: M0处理完成，发送ok
- `m0_paper_change_complete()`: 换纸完成

这个修正确保了M0和换纸的正确分离，提高了系统的响应性和可靠性。
