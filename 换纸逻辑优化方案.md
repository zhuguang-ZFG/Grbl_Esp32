# 换纸逻辑优化方案

## 🔍 发现的关键问题

### 1. ❌ **出纸长度严重不足**

#### 原始问题
```cpp
// 错误设置
if (paper_ctrl.step_counter < 100) {  // 仅100步
    // 出纸操作...
}

// 计算结果：100步 ÷ 80步/mm = 1.25mm
// A4实际需要：297mm
// 缺少：295.75mm (99.6%的纸张未送出！)
```

#### 修复方案
```cpp
// 优化设置
#define PAPER_EJECT_STEPS          23800    // steps - A4长度: 297mm × 80步/mm  
#define PAPER_EJECT_INTERVAL_US     1000      // μs - 1ms间隔，提高速度

// 修复代码
if (paper_ctrl.step_counter < PAPER_EJECT_STEPS) {  // 23800步 = 297mm
    generate_motor_step(BIT_PANEL_MOTOR_STEP, BIT_PANEL_MOTOR_DIR, true);
    paper_ctrl.step_counter++;
    delayMicroseconds(PAPER_EJECT_INTERVAL_US); // 1ms间隔，提高速度
}
```

## 🎯 具体优化措施

### 1. **出纸逻辑优化**

#### A. 步数计算优化
```
A4长度: 297mm
电机精度: 80步/mm
所需步数: 297mm × 80步/mm = 23,760步
安全余量: +40步 (0.5mm)
最终步数: 23,800步
```

#### B. 速度优化
```
原始速度: 2ms/步 = 500步/秒 = 6.25mm/秒
优化速度: 1ms/步 = 1000步/秒 = 12.5mm/秒
提升幅度: 100%速度提升
```

### 2. **精确定位优化**

#### A. 延迟优化
```cpp
// 原始：3ms延迟 → 优化：2ms延迟
delayMicroseconds(2000);  // 提高33%响应速度
```

#### B. 注释完善
```cpp
// Forward 32 steps - 前进32步(0.4mm)，确保纸张完全到位
// Back 2 steps - 后退2步(0.025mm)，微调纸张精确位置
```
