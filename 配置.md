
## 写字机硬件与固件配置说明
### 1. 主控与固件
- **主控芯片**: ESP32‑WROOM‑32（ESP32‑D0WDQ6，晶振 40MHz）  
- **固件**: Grbl_ESP32（3 轴写字机，基于 `custom_3axis_hr4988.h` 机器配置）  
- **开机信息**（串口上电日志中可见）：  
  - 固件版本与 SDK：  
    - `Grbl_ESP32 Ver ...`  
    - `Compiled with ESP32 SDK:...`  
  - **芯片唯一 ID**（用于授权/加密绑定）：  
    - 由 `machine_init()` 打印：  
      - `[ESP32ID] ChipID (efuse MAC) = %08X%08X`  
    - 建议上位机在初始化阶段读取并缓存此 ID，后续可用作程序加密或授权判断的硬件指纹。
---
### 2. 轴向步进电机配置
- **X 轴**  
  - STEP: `GPIO2`  
  - DIR : `GPIO15`  
- **Y 轴（Y1+Y2 两个电机并联，硬件共用 STEP/DIR）**  
  - STEP: `GPIO13`  
  - DIR : `GPIO12`  
- **Z 轴（抬笔）**  
  - STEP: `GPIO14` (MTMS)  
  - DIR : `GPIO27`  
> 说明：Y 轴是双电机并联接在同一套 STEP/DIR 上；Z 轴只负责抬笔。  
> **Z 轴开机默认在原点**：机械上为抬笔方向极限，系统认为 **Z=0**；落笔为 Z 正向（如 Z30）。
---
### 3. HR4988 驱动配置
- **参考电压**: 四个 HR4988 共用 `REF`，接 `GPIO25`（仅作参考电压输入，固件不切换此脚）
- **使能脚（nENABLE，低电平=使能）**  
  - **主 XYZ 三轴**: `GPIO4`  
  - **拾落电机 + 进纸器电机**: `GPIO26`  
  - **面板电机**: 通过 74HC595 的 `Q1`（QB，对应逻辑引脚 `I2SO(1)`）
---
### 4. 74HC595D 扩展（换纸系统）
#### 4.1 控制信号连接（ESP32 → 74HC595D）
- **SI（数据输入，Pin 14）**: `GPIO21`  
- **SRCLK（移位时钟，Pin 11）**: `GPIO16`  
- **RCLK（存储时钟/锁存，Pin 12）**: `GPIO17`  
- **SRCLR（复位，Pin 10）**: 接 VCC（高电平，不复位）  
- **OE（输出使能，Pin 13）**: 接 GND（低电平=输出使能）
#### 4.2 74HC595D 输出位定义（Q0..Q7）
- **Q0 (`Pin 15`)**: 一键换纸按钮彩灯（固件已按状态驱动）  
  - HIGH = 灯灭，LOW = 灯亮  
  - **状态含义**：  
    - **常亮**：空闲且有纸（就绪，可按键换纸）或运行中有纸  
    - **慢闪（约 0.5s 周期）**：空闲且无纸（提示加纸）  
    - **快闪（约 80ms 周期）**：一键换纸流程执行中  
- **Q1 (`Pin 1`)**: 面板电机 HR4988 使能（nENABLE）  
  - LOW = 使能  
  - HIGH = 禁用  
- **Q2 (`Pin 2`)**: 拾落电机 DIR（方向，接 HR4988 Pin 19）  
- **Q3 (`Pin 3`)**: 拾落电机 STEP（步进，接 HR4988 Pin 16）  
- **Q4 (`Pin 4`)**: 面板电机 DIR（方向，接 HR4988 Pin 19）  
- **Q5 (`Pin 5`)**: 面板电机 STEP（步进，接 HR4988 Pin 16）  
- **Q6 (`Pin 6`)**: 进纸器电机 DIR（方向，接 HR4988 Pin 19）  
- **Q7 (`Pin 7`)**: 进纸器电机 STEP（步进，接 HR4988 Pin 16）
> 固件中对应逻辑引脚为：  
> - `I2SO(0)~I2SO(7)` 分别对应 Q0~Q7  
> - 拾落：DIR=`I2SO(2)`，STEP=`I2SO(3)`  
> - 面板：DIR=`I2SO(4)`，STEP=`I2SO(5)`  
> - 进纸：DIR=`I2SO(6)`，STEP=`I2SO(7)`
---
### 5. 传感器与按钮
- **纸张检测传感器**: `GPIO34`  
  - 逻辑：**LOW = 有纸，HIGH = 无纸**  
  - 当前配置中**不再映射为 Macro0 按钮**，故不出现在 `?` 指令的 `|Pn:` 中  
  - 状态通过专用命令查询：`M701`、`M704` 或 `[ESP901]`
- **一键换纸按钮**: `GPIO35`  
  - 低电平 = 按下（需要外接下拉或合理上拉电路）  
  - 映射为 `MACRO_BUTTON_0_PIN`（Macro0），并通过 `INVERT_CONTROL_PIN_MASK` 设为**低电平有效**  
  - **软件去抖**：在 `Custom/paper_system.cpp::user_defined_macro()` 中实现  
    - 仅在 `sys.state == Idle` 时响应按键  
    - 两次有效触发之间至少间隔 **500ms**（`millis()` 节流），防止抖动和误连按  
    - 每次有效触发会打印：  
      - `[PaperBtn] Triggered (GPIO35=x), queuing [ESP910] auto-change...`  
    - 按键触发的换纸实际是向输入缓冲注入一条 `[ESP910]`，走与 Web 指令完全一致的自动换纸流程。
---
### 6. 换纸相关命令
#### 6.1 推荐：专用换纸命令（[ESP9xx]）
这些命令通过 Web/串口统一入口（非纯 GCode），推荐用于调试和点动控制：
- **查询纸张+电机状态**  
  ```text
  [ESP901]
返回类似：Paper=OK/No MotorEn=On/Off

- **拾落 / 面板 / 进纸器电机点动**（可带步数，默认 200 步，便于单独调试）  
  ```text
  [ESP911]        ; 拾落电机 200 步
  [ESP911 50]     ; 拾落电机 50 步
  [ESP912]        ; 面板电机 200 步
  [ESP912 100]    ; 面板电机 100 步
  [ESP913]        ; 进纸器电机 200 步
  [ESP913 80]     ; 进纸器电机 80 步
  ```
- **仅使能换纸驱动（不动作）**，便于先设方向再点动：  
  ```text
  [ESP930]
  ```
  执行后 MotorEn=On，可用 `M64 P1`/`M65 P1` 等设方向，再用 `[ESP911]`/`[ESP912]`/`[ESP913]` 点动。

串口或 Web 控制台里原样输入（包括方括号），回车即可，例如发 [ESP901] 会回复一行状态和 ok。

6.1.2 一键自动换纸（推荐）

- **Web/串口命令**：  
  ```text
  [ESP910]
  ```  
  或按下一键换纸按钮（GPIO35，已做硬件+软件去抖，单次按下只触发一次 [ESP910]）。

- **自动换纸整体流程概述**（步数和方向可在 `custom_3axis_hr4988.h` 中调整宏）：  
  1. **使能换纸相关驱动（互斥两组）**  
     - 切换 I2S 为静态模式（passthrough）  
     - 面板组（Q1 低）：只在需要时单独使能  
     - 拾落+进纸组（GPIO26 低）：只在需要时单独使能  
  2. **面板电机弹出旧纸**  
     - DIR = `PANEL_DIR_EJECT`  
     - 运行 `PANEL_EJECT_STEPS` 步（约等于一张 A4 长度 + 余量，可调）  
  3. **进纸器送入新纸直到纸张传感器触发**  
     - DIR = `FEEDER_DIR_FORWARD`  
     - 单步前进，配合 `paper_sensor_stable()`（3 次采样、100µs 间隔）判断“有纸”，上限 `FEEDER_FIND_STEPS_MAX`  
  4. **松开拾落电机**  
     - DIR = `CLAMP_DIR_RELEASE`  
     - STEP = `CLAMP_TOGGLE_STEPS`（默认约 470 步，可调）  
  5. **进纸器继续送入一段（约 8cm）**  
     - 继续 DIR = `FEEDER_DIR_FORWARD`，前进 `FEEDER_EXTRA_STEPS` 步  
  6. **再次压紧拾落电机**  
     - DIR = `CLAMP_DIR_CLAMP`  
     - STEP = `CLAMP_TOGGLE_STEPS`  
  7. **面板电机快速送纸，直到传感器“看不到纸”**  
     - 仅面板电机使能，DIR = `PANEL_DIR_FEED`  
     - 单步前进，配合 `paper_sensor_stable()` 判断“无纸”，上限 `PANEL_FAST_STEPS_MAX`  
  8. **面板电机按 `PANEL_DIR_REVERSE` 回找传感器**  
     - 仅面板电机使能，DIR = `PANEL_DIR_REVERSE`  
     - 单步回退，直到“再次感应到纸”或达到 `PANEL_BACK_STEPS_MAX` 上限  
  9. **面板电机再向送纸方向走固定步数，完成最终对位**  
     - DIR = `PANEL_DIR_FEED`  
     - STEP = `PANEL_FINAL_STEPS`  
  10. **关闭换纸相关电机使能**  
      - 拉高 Q1 和 GPIO26，防止三路纸路电机长时间发热  
      - 串口输出 `[PaperAuto] All steps completed successfully!` 及一行 **`[PaperStatus] N`**（见下）。

- **换纸结束状态码 `[PaperStatus] N`**（上位机可解析此行做分支）：  
  | N | 含义 |
  |---|------|
  | 0 | 成功完成全流程 |
  | 1 | 保留（未使用） |
  | 2 | 进纸阶段在限定步数内未触发传感器，已中止 |
  | 3 | 流程跑完，但第 7 步回找后传感器未稳定（纸可能未到位，仅警告） |

- **开始队列书写前出一次旧纸**：  
  - 设计上：上位机在**发送第一页 GCode 之前**先发 **`M721`**，固件会自动执行一次完整换纸（出旧纸 + 进第一张并对位），再回 `ok`。  
  - **若上位机不会发送 M721**（如奎享雕刻当前不支持）：需在开始队列前**手动**执行一次 **`[ESP910]`** 或按**一键换纸键**，效果相同；然后再由上位机发送第一页、第二页…（每页末尾 M30 自动换纸）。

- **可调参数汇总**（均在 `custom_3axis_hr4988.h`）：  
  - 步数：`PANEL_EJECT_STEPS`、`PANEL_FAST_STEPS_MAX`、`PANEL_BACK_STEPS_MAX`、`PANEL_FINAL_STEPS`、`FEEDER_FIND_STEPS_MAX`、`FEEDER_EXTRA_STEPS`、`CLAMP_TOGGLE_STEPS`  
  - 步进脉宽（μs）：`PAPER_RAMP_STEPS`、`PAPER_RAMP_HI_US`/`PAPER_RAMP_LO_US`、`PAPER_NORMAL_HI_US`/`PAPER_NORMAL_LO_US`、`PAPER_CLAMP_HI_US`/`PAPER_CLAMP_LO_US`  
  - 方向：`PANEL_DIR_FEED`、`PANEL_DIR_EJECT`、`PANEL_DIR_REVERSE`、`FEEDER_DIR_FORWARD`、`CLAMP_DIR_*`  
  - **Z 轴点动限速**：`JOG_Z_MAX_FEED_MM_PER_MIN`（默认 1000 mm/min）。点动含 Z 向移动时（抬落笔）进给会被限制到此值，避免 $J= 用较大 F（如 2500）时 Z 过快；仅 XY 点动不受限。

6.3 换纸电机步进时序（卡顿/带载不足时可调）
- **脉宽由头文件宏控制**（`custom_3axis_hr4988.h`）：  
  - 面板/进纸器：起步 `PAPER_RAMP_STEPS` 步内为 `PAPER_RAMP_HI_US`/`PAPER_RAMP_LO_US`（默认 400µs），之后为 `PAPER_NORMAL_HI_US`/`PAPER_NORMAL_LO_US`（默认 150µs）。  
  - 拾落电机：`PAPER_CLAMP_HI_US`/`PAPER_CLAMP_LO_US`（默认 2000µs）。  
- 若进纸器**仍卡顿**：优先检查 REF 电流、供电与接线；必要时将 `PAPER_NORMAL_*_US` 或 `PAPER_RAMP_*_US` 略增大。  
- **REF 直连 GPIO25（DAC）**：`PAPER_DRIVER_REF_DAC` 默认 100（≈1.3V）；直连时建议 80–150，发烫则调小。

6.2.5 M30 / M02 自动换纸（分页写字，“每页一个文件”模式）

- **设计目标**：一页 GCode 执行到**程序结束码**时，固件先同步完本页所有运动，再**自动换纸**，换纸结束后才回复 `ok`。  
- **触发条件**：固件对 **`M30`** 和 **`M02`** 都会执行换纸（二者在 Grbl 里均会调用 `user_m30()`）。  
  - 若上位机（如奎享雕刻）在**每页/每文件结束时没有下发 M30 或 M02**，固件不会执行换纸，这是“队列结束没换纸”的常见原因。  
- **固件实现要点**：  
  - `GCode.cpp` 中，M30→`CompletedM30`、M02→`CompletedM2`，在执行 program flow 时都会：  
    1. `protocol_buffer_synchronize()`，确保本页运动执行完；  
    2. 重置部分 modal 状态；  
    3. 调用 `user_m30()`（`Custom/paper_system.cpp`）；  
  - `user_m30()` 仅在 Idle 且纸张系统已配置时调用 `paper_auto_change()`，换纸结束后才返回，随后该行才回 `ok`。  
- **奎享雕刻等上位机**：  
  - 若每页末尾是 **`G90 G0 X0 Y0`**（回原点），固件会在此行执行完后自动换纸，**无需**再加 M30（见 6.2.6）。  
  - 若每页末尾**不是**回原点，则**必须**在每页文件末尾包含 **`M30`** 或 **`M02`**，固件才会换纸。若软件不会自动加 M30，可：  
    1. 在生成/导出每页 GCode 时**手动在文件末尾加一行 `M30`**；或  
    2. 在奎享中查找是否有“每文件结束发送 M30”或“程序结束符”类设置并开启。  
  - 可用串口监控查看：每页发送完毕后，是否有一条以 `M30` 或 `M02` 结尾的指令；若没有，说明上位机未发程序结束码，需按上面方式补上。  
- **正确流程**：每页文件末尾有 M30（或 M02）→ 固件执行完本页并换纸 → 回 `ok` → 上位机再发下一页。

6.2.6 回原点后自动换纸（适配奎享雕刻等不发 M30 的上位机）

- **触发方式**：固件在检测到**回原点**指令并执行完该运动后，会自动执行一次换纸（与 M30 相同逻辑）。  
- **会触发换纸的指令**：  
  1. **G28**、**G30**（回预设原点）；  
  2. **G0 X0 Y0**（或 `G90 G0 X0 Y0`）：快速移动到当前工件坐标系 XY 零点。  
- **奎享雕刻**：若每页末尾是 **`G90 G0 X0 Y0`** 而不是 M30，固件会在此行运动完成后自动换纸，**无需**在每页文件里加 M30。  
- **实现**：在 `GCode.cpp` 中，**仅当本行实际执行了**回原点（G28/G30 或本行有 G0 且目标为工作坐标 (0,0)）时，才先 `protocol_buffer_synchronize()` 再 `user_m30()`。必须用“本行是否执行了 G0”标志，否则下一页首行（如 G21）会因 modal 仍为 Seek、position 仍为 (0,0) 而误触发换纸（表现为“队列开始后一直在换纸”）。工作坐标 (0,0) 容差 0.01mm。

- **关于奎享末尾行 `G90G0 X0Y0`**：  
  - Grbl 按“字母+数值”分词，`G90G0` 会解析为 **G90** 和 **G0**，`X0Y0` 会解析为 **X0** 和 **Y0**，因此理论上会触发“回原点换纸”。  
  - 若实际未换纸，可能原因：  
    1. **user_m30() 内部被跳过**：仅在 `sys.state == Idle` 且纸张系统已配置（`PAPER_SENSOR_PIN != 255`）时才执行换纸；否则串口会看到 `[PaperM30] Skipped auto-change: ...`。  
    2. **上位机未等本行 ok**：若奎享在发完 `G90G0 X0Y0` 后立刻发下一页或断开，固件仍会执行换纸，但可能看起来像“没反应”（可观察串口是否先出 `ok` 再出下一页）。  
  - **调试**：在机器头文件或 `Config.h` 中定义 **`#define DEBUG_PAPER_ORIGIN`** 后重新编译，串口会在检测到回原点时打印 `[PaperOrigin] Return-to-origin detected, syncing then paper change`，用于确认该行是否被识别。

- **每页 GCode 格式约定**（奎享 / 导出多页时建议统一）：  
  - **页首**：`G21`、`G90`，以及 `F1500`、`G1 G90 Z0.0`、`G4 P0.2` 等本机需要的初始化。  
  - **页尾**：**`G90 G0 X0 Y0`**（或 `G90G0 X0Y0`）。固件执行到该行会先回原点，再自动换纸，然后回 `ok`。  
  - 多页时：每页都是「页首 + 本页书写 + 页尾」，下一页紧接下一段「页首 + 书写 + 页尾」；页尾的 `G90 G0 X0 Y0` 即分页符，遇到一次换纸一次。

6.2.7 开始写字前自动换纸（无需 M721 / 手动键）

- **行为**：上电（或 Grbl 复位）后，固件**仅当第一次**执行 **G0 X0 Y0 Z0**（回原点，含 X/Y/Z 且目标为 0）时，先同步缓冲区、执行一次完整换纸（出旧纸 + 进第一张并对位）、延时 250ms，再执行该行；串口会打印 `[Paper] 1st page`。纯抬笔/落笔（如 G1 Z0、G1 Z30）或仅 XY 的 G0/G1 不触发。
- **效果**：队列开头若有 **G0 X0 Y0 Z0**，点“开始”后会自动换纸再回原点；第一页末尾用 `G90 G0 X0 Y0` 触发换纸，之后每页同理。若队列开头没有 G0 X0 Y0 Z0，需手动 [ESP910] 或 M721 做第一页前换纸。
- **同一上电周期内跑多个队列**：仅第一次遇到 G0 X0 Y0 Z0 时换纸；之后若再跑第二队列且开头无 G0 X0 Y0 Z0，需手动 [ESP910] 或 M721。
- **说明**：上述第一页/页末换纸及 Z 轴（开机 Z=0、换纸后 Z=0）逻辑结构已用他人程序验证，无问题。

6.2.8 M721「队列开始」：第一页前换纸（可选）

## 8. 故障排除与修复记录

### 8.1 问题1：GPIO34 纸张传感器无法正常工作

**现象**：固件启动后执行 M701 或 M704 命令，显示 "Paper=OK MotorEn=On"，无论是否有纸都输出相同结果；传感器状态不变。

**原因**：GPIO34 从未使用 `pinMode(GPIO_NUM_34, INPUT)` 初始化，导致无法进行外部电压的正确采样，GPIO 引脚浮空或保持重置时的默认状态，固件逻辑无法感知实际电压变化。

**解决方案**（已实施 / 2026-03-12）：
- 在 `paper_system_init()` 函数中添加：
  ```cpp
  pinMode((int)PAPER_SENSOR_PIN, INPUT);
  int initial_value = digitalRead(PAPER_SENSOR_PIN);
  ```
- 同时读取并输出初始值用于启动时调试，确保传感器连接正确

**验证方法**：
1. 重新编译 & 上传固件
2. 执行 M701 或 M704 命令，应输出 GPIO34 原始值（0 或 1）
3. 插入/移除纸张，重新执行 M704，GPIO 值应改变
4. 若值仍不变，检查 GPIO34 硬件连接和传感器本身

### 8.2 问题2：自动升级 (Auto Download) 失败，无法进入 Bootloader

**现象**：上传固件时 PlatformIO 报错或卡住，需要手动按 IO0 + RESET 才能进入 bootloader；自动进入 bootloader 的机制失效。

**原因**：原始代码在 `paper_system_init()` 中过早调用 `i2s_out_set_passthrough()`，在系统启动的早期就初始化了 I2S 外设。**GPIO16 和 GPIO17 是 I2S 的 SRCLK 和 RCLK 脚线，在某些 ESP32 芯片版本上与内部 Flash SPI 或 USB 接口共享**；过早初始化 I2S 可能干扰这些脚线的状态，导致自动 bootloader 序列失败。

**解决方案**（已实施 / 2026-03-12）：
- 延迟 I2S passthrough 模式的设置，从启动时移到**第一次真正需要电机操作时**
- 具体改动：
  - `paper_system_init()` 现在**仅初始化 GPIO34（传感器）和 GPIO25（DAC）**，完全不涉及 I2S 操作
  - `paper_enable_drivers()` 中添加一次性标志，仅第一次调用时执行 `i2s_out_set_passthrough()`，之后跳过
  - I2S 硬件初始化仍由 `Grbl.cpp` 的全局初始化流程负责（不重复）

**原理**：  
系统启动时 `Grbl.cpp` 已完整初始化 I2S 硬件，但不必立刻切换 passthrough 模式。将 passthrough 模式设置推迟到执行电机命令时，可避免在 bootloader 决策期间对 GPIO16/17 进行额外处理。

**验证方法**：
1. 重新编译并尝试**自动上传**（不需要手动按按键）
2. 自动升级应恢复正常
3. 如仍有问题，用 esptool.py 手动检查是否真的进入 bootloader

**关键代码位置**：
- [PaperSystem.cpp](Grbl_Esp32/src/PaperSystem.cpp) 第 43–55 行（paper_system_init）
- [PaperSystem.cpp](Grbl_Esp32/src/PaperSystem.cpp) 第 84–101 行（paper_enable_drivers）

### 8.3 问题3：写完最后一张没排出去 / 刚开始写时 Z 轴卡顿

**现象 1：队列最后一页写完后，最后那张纸没有排出去**  
- 先看串口：最后一页的 `G90 G0 X0 Y0` 之后是否出现 **`[PaperAuto-1] Ejecting old paper (12000 steps)...`**。  
  - **有**：说明换纸已触发，Step1 在执行。若纸仍没出来，多半是**步数不够**或**方向反了**。在 `custom_3axis_hr4988.h` 里：  
    - 加大 **`PANEL_EJECT_STEPS`**（默认 12000），例如改为 15000～18000 试到能完整排出；  
    - 若纸往反方向走，把 **`PANEL_DIR_EJECT`** 取反（`true`↔`false`）。  
  - **没有**：说明最后一页没有触发换纸，检查该页末尾是否真有 `G90 G0 X0 Y0`。

**现象 2：刚开始写时 Z 轴卡顿一下**  
- 固件已在“第一页换纸”结束后加 **250ms 延时** 再执行第一条书写；若确认卡顿与换纸无关，见下条。

**Z 轴卡顿 / 方向反了（可逐项排除）**  
- **“先反一下再正”**：固件已把 **$Stepper/Direction/Delay** 默认设为 **15µs**（`STEP_PULSE_DELAY`）；若仍卡顿，可试 **20～25µs**（串口发 `$` 查当前值，改 Stepper/Direction/Delay 后 `$S` 保存，无需重烧）。  
- **整根 Z 方向反了**（抬笔/落笔与 GCode 相反）：用 **$3=4** 反相 Z 轴方向（bit2=Z），再 **$S** 保存。  
- **接线**：Z 轴 STEP=GPIO14(MTMS)、DIR=GPIO27；确认无虚接、与 74HC595/I2S 无串扰。  
- **供电**：Z 与 XY/换纸电机共电源时，换纸瞬间电压跌落可能导致 Z 顿一下；可试单独供电或加大滤波。  
- **参数**：`$120`（Z 加速度）、`$122`（Z 最大速率）过大可导致起步过猛；可适当减小试机。  
- **点动限速**：Z 点动已用 `JOG_Z_MAX_FEED_MM_PER_MIN` 限速；若卡顿只在 GCode 时出现，可对比同速度点动是否也卡。  

**固件侧已核对（尚未发现可解释“卡顿”的明显错误）**  
- **Stepper ISR**：新 segment 加载时先更新 `dir_outbits`，下一拍再发步进；块边界处未见“多发一步/反方向一步”的逻辑错误。  
- **可能现象**：若卡顿是“起步那一下特别慢，然后正常”，多半是**加速段**：从静止到 F3000 的前几毫秒，Planner 会生成步数很少、周期较长的 segment，看起来像顿一下再走。可试把 **$120（Z 加速度）** 适当加大（例如 1200～1500），让加速段更短，看观感是否改善。  
- **进一步排查**：用示波器看 Z 的 STEP/DIR 是否有一次反向脉冲、或首步脉宽异常；或临时在 Z 步进时拉高某 GPIO 用 LED 观察节奏，区分“多/错一步”与“起步慢”。

- 若后续找到原因，可把结论补写在本节或 8.4 调试记录中。

### 8.4 调试与未来改进

若后续发现类似的初期化顺序问题，优先原则：
1. **硬件初始化** → 传感器、DAC、简单 GPIO（早期）
2. **复杂外设初始化**（I2S、DMA、中断）→  由全局框架统一管理
3. **功能模式开启**（passthrough、PWM、特定任务）→ 延迟到实际需要时

此原则可有效避免 bootloader 干扰和启动时序问题。
