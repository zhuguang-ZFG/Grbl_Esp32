## 写字机硬件与固件配置说明
### 1. 主控与固件
- **主控芯片**: ESP32‑WROOM‑32（ESP32‑D0WDQ6，晶振 40MHz）  
- **固件**: Grbl_ESP32（3 轴写字机，基于 `custom_3axis_hr4988.h` 机器配置）
---
### 2. 轴向步进电机配置
- **X 轴**  
  - STEP: `GPIO2`  
  - DIR : `GPIO15`  
- **Y 轴（Y1+Y2 两个电机并联，硬件共用 STEP/DIR）**  
  - STEP: `GPIO13`  
  - DIR : `GPIO12`  
- **Z 轴（抬笔）**  
  - STEP: `GPIO14` (MTMS)  
  - DIR : `GPIO27`  
> 说明：Y 轴是双电机并联接在同一套 STEP/DIR 上；Z 轴只负责抬笔。
---
### 3. HR4988 驱动配置
- **参考电压**: 四个 HR4988 共用 `REF`，接 `GPIO25`（仅作参考电压输入，固件不切换此脚）
- **使能脚（nENABLE，低电平=使能）**  
  - **主 XYZ 三轴**: `GPIO4`  
  - **拾落电机 + 进纸器电机**: `GPIO26`  
  - **面板电机**: 通过 74HC595 的 `Q1`（QB，对应逻辑引脚 `I2SO(1)`）
---
### 4. 74HC595D 扩展（换纸系统）
#### 4.1 控制信号连接（ESP32 → 74HC595D）
- **SI（数据输入，Pin 14）**: `GPIO21`  
- **SRCLK（移位时钟，Pin 11）**: `GPIO16`  
- **RCLK（存储时钟/锁存，Pin 12）**: `GPIO17`  
- **SRCLR（复位，Pin 10）**: 接 VCC（高电平，不复位）  
- **OE（输出使能，Pin 13）**: 接 GND（低电平=输出使能）
#### 4.2 74HC595D 输出位定义（Q0..Q7）
- **Q0 (`Pin 15`)**: 一键换纸按钮指示灯  
  - HIGH = 灯灭  
  - LOW  = 灯亮  
- **Q1 (`Pin 1`)**: 面板电机 HR4988 使能（nENABLE）  
  - LOW = 使能  
  - HIGH = 禁用  
- **Q2 (`Pin 2`)**: 拾落电机 DIR（方向，接 HR4988 Pin 19）  
- **Q3 (`Pin 3`)**: 拾落电机 STEP（步进，接 HR4988 Pin 16）  
- **Q4 (`Pin 4`)**: 面板电机 DIR（方向，接 HR4988 Pin 19）  
- **Q5 (`Pin 5`)**: 面板电机 STEP（步进，接 HR4988 Pin 16）  
- **Q6 (`Pin 6`)**: 进纸器电机 DIR（方向，接 HR4988 Pin 19）  
- **Q7 (`Pin 7`)**: 进纸器电机 STEP（步进，接 HR4988 Pin 16）
> 固件中对应逻辑引脚为：  
> - `I2SO(0)~I2SO(7)` 分别对应 Q0~Q7  
> - 拾落：DIR=`I2SO(2)`，STEP=`I2SO(3)`  
> - 面板：DIR=`I2SO(4)`，STEP=`I2SO(5)`  
> - 进纸：DIR=`I2SO(6)`，STEP=`I2SO(7)`
---
### 5. 传感器与按钮
- **纸张检测传感器**: `GPIO34`  
  - 逻辑：**LOW = 有纸，HIGH = 无纸**  
  - 在 Grbl 状态查询 `?` 中映射为 Macro0：  
    - 状态行里 `|Pn:0` 出现 → “无纸”  
    - 状态行里 `Pn:` 中没有 `0` → “有纸”
- **一键换纸按钮**: `GPIO35`  
  - 低电平 = 按下（需要外接下拉或合理上拉电路）
---
### 6. 换纸相关命令
#### 6.1 推荐：专用换纸命令（[ESP9xx]）
这些命令通过 Web/串口统一入口（非纯 GCode），推荐用于调试和点动控制：
- **查询纸张+电机状态**  
  ```text
  [ESP901]
返回类似：Paper=OK/No MotorEn=On/Off

- **拾落 / 面板 / 进纸器电机点动**（可带步数，默认 200 步，便于单独调试）  
  ```text
  [ESP911]        ; 拾落电机 200 步
  [ESP911 50]     ; 拾落电机 50 步
  [ESP912]        ; 面板电机 200 步
  [ESP912 100]    ; 面板电机 100 步
  [ESP913]        ; 进纸器电机 200 步
  [ESP913 80]     ; 进纸器电机 80 步
  ```
- **仅使能换纸驱动（不动作）**，便于先设方向再点动：  
  ```text
  [ESP930]
  ```
  执行后 MotorEn=On，可用 `M64 P1`/`M65 P1` 等设方向，再用 `[ESP911]`/`[ESP912]`/`[ESP913]` 点动。

串口或 Web 控制台里原样输入（包括方括号），回车即可，例如发 [ESP901] 会回复一行状态和 ok。

6.1.2 一键自动换纸（推荐）

[ESP910]
执行一整套自动换纸流程，具体步骤如下（步数和方向可在 custom_3axis_hr4988.h 中调整宏）：

1. 使能换纸相关驱动  
   - 切换 I2S 为静态模式（passthrough）  
   - 拉低 74HC595 Q1（面板电机 nENABLE）  
   - 拉低 GPIO26（拾落电机 + 进纸器电机 nENABLE）

2. 面板电机运动，弹出旧纸  
   - 面板 DIR 设为“弹出旧纸”方向（PANEL_DIR_EJECT）  
   - 运行 PANEL_EJECT_STEPS 步（约等于一张 A4 长度 + 余量，可调）

3. 进纸器送入新纸直到纸张传感器触发  
   - 进纸器 DIR 设为“送纸进入机器”方向（FEEDER_DIR_FORWARD）  
   - 单步前进，持续检测纸张传感器，直到检测到纸或达到 FEEDER_FIND_STEPS_MAX 上限

4. 松开拾落电机  
   - 拾落电机 DIR 设为“松开纸张”方向（CLAMP_DIR_RELEASE）  
   - 运行 CLAMP_TOGGLE_STEPS 步完成一次完整“抬纸”动作

5. 进纸器继续送入一段（约 5cm）  
   - 继续按 FEEDER_DIR_FORWARD 方向前进 FEEDER_EXTRA_STEPS 步，让新纸再进入一些

6. 再次压紧拾落电机  
   - 拾落电机 DIR 设为“压紧纸张”方向（CLAMP_DIR_CLAMP）  
   - 再运行 CLAMP_TOGGLE_STEPS 步完成压纸动作

7. 面板电机 + 进纸器快速送纸，直到传感器“看不到纸”  
   - 面板 DIR 设为“送纸方向”（PANEL_DIR_FEED），进纸器保持 FEEDER_DIR_FORWARD  
   - 两电机 1:1 交替单步前进：每次循环各走 1 步  
   - 一直持续到纸张传感器检测不到纸，或达到 PANEL_FAST_STEPS_MAX 上限

8. 面板电机反向回找感应点  
   - 面板 DIR 反向（与送纸方向相反，使用 PANEL_DIR_EJECT 或根据实际调整）  
   - 单步后退，直到纸张传感器再次“感应到纸”，或达到 PANEL_BACK_STEPS_MAX 上限

9. 面板电机再向送纸方向走固定步数，完成最终对位  
   - 面板 DIR 再次设为 PANEL_DIR_FEED  
   - 前进 PANEL_FINAL_STEPS 步，作为最后的对位修正  
   - 流程结束后串口会输出 “Paper auto: done”

6.3 换纸电机步进时序（卡顿/带载不足时可调）
- **面板电机**：约 150µs 高/低，步频高、运行顺滑。
- **进纸器电机**：与面板电机相同，约 150µs 高/低（已按你要求改成和面板配置一致），主要通过 REF 电流大小决定带载能力。
- **拾落电机**：2ms 高/低。
- 若进纸器**仍卡顿**：优先检查硬件——进纸器对应 HR4988 的 **REF 电流**是否足够（可略高于面板）、电机供电电压、STEP/DIR/EN 接线与共地；必要时再整体把纸路电机步进延时适当放大（在 `PaperSystem.cpp` 中调整 150µs/2ms 这两组值）。
- **REF 直连 GPIO25（DAC，无分压）**：固件启动时输出 DAC 默认 100（≈1.3V）。直连时建议 `PAPER_DRIVER_REF_DAC` 保持在 80–150；无力可略调大，发烫则调小。
