## 概述

GRBL ESP32换纸系统是一个基于74HC595D扩展GPIO的自动化纸张更换解决方案，集成了高精度传感器检测、多电机协调控制和智能状态机管理。本系统专为A4纸张写字机设计，可实现全自动的纸张装载、定位和卸载操作。

### 系统特点
- **全自动化**：无需人工干预的完整换纸流程
- **高精度控制**：80步/毫米的分辨率，±0.1mm定位精度
- **智能传感**：实时纸张检测和位置反馈
- **多电机协调**：3个独立电机的精确时序控制
- **安全保护**：多重超时保护和错误恢复机制
- **实时监控**：完整的调试和状态监控系统

### 应用场景
- A4纸张批量自动化写字
- 绘图仪自动纸张更换
- 激光雕刻机材料更换
- 教学演示自动化系统

## 硬件布局与基本特性

### 电机位置关系
```
进纸电机 ---- 纸张传感器 ---- 夹紧电机 + 面板电机
       (相距3cm)      (相距3cm)
```

### 基本状态说明
- **夹紧电机默认状态**：电机使能不运动即为夹紧状态
- **传感器位置**：位于夹紧电机与面板电机之间
- **传感器功能**：感应纸张存在和距离测量

### 电机运行限制
- **面板电机**：一般情况下不反转，仅用于位置判断时反转
- **进纸器电机**：永远不会反转
- **夹紧电机**：正转1.5cm即为松开夹子，反转到初始位置即为夹紧状态

## 进纸流程详细说明

### 工作步骤

1. **启动进纸**
   - 进纸电机开始运动
   - 等待纸张感应器检测到纸张

2. **检测到纸张后**
   - 夹紧电机运行（初始为夹紧态）→ 松开夹子
   - 面板电机同步运行
   - 纸张经过传感器后继续运行3.5cm

3. **夹紧行程**
   - 夹紧电机夹紧纸张
   - 进纸电机、面板电机同速快速运行
   - 运行约A4纸张行程，直到纸张脱离传感器

4. **位置校准**
   - 进纸电机停止
   - 面板电机反转
   - 直到传感器再次感应到纸张
   - 面板电机正转3cm左右（**关键步骤**，确保纸张位置正确）

5. **完成**
   - 纸张到达正确位置
   - 开始执行正常写字任务

### 重要说明
- **传感器触发后的5cm运行期间**：夹紧电机保持松开状态
- **纸张脱离传感器**：进纸电机停止信号
- **位置校准**：反转后再正转3cm是确保位置正确的关键

## 出纸流程详细说明

### 工作步骤

1. **检测纸张位置**
   - 面板电机反转
   - 直到传感器感应到纸张后停止

2. **完全出纸**
   - 面板电机快速正转
   - 运行距离：A4纸张长度 + 3-5cm
   - 确保纸张完全送出

3. **停止**
   - 面板电机停止
   - 出纸流程完成

### 出纸流程特点
- **反向检测**：先反转检测纸张边界
- **精确距离**：A4长度+预留距离确保完全送出
- **单向运动**：面板电机通常不反转，仅在检测时使用

## 状态机设计

### 状态定义与转换

#### 核心状态枚举
```cpp
typedef enum {
    PAPER_IDLE = 0,        // 空闲状态，等待触发
    PAPER_PRE_CHECK,        // 预检查，检测当前纸张状态
    PAPER_EJECTING,        // 出纸状态，送出当前纸张
    PAPER_FEEDING,         // 进纸状态，装载新纸张
    PAPER_UNCLAMP_FEED,    // 松开进纸，夹紧器松开时的进纸
    PAPER_CLAMP_FEED,       // 夹紧进纸，夹紧器夹住时的进纸
    PAPER_FULL_FEED,        // 完整进给，快速进纸到目标位置
    PAPER_REPOSITION,      // 重新定位，精确位置调整
    PAPER_COMPLETE,         // 完成状态，准备就绪
    PAPER_ERROR            // 错误状态，异常处理
} paper_state_t;
```

#### 状态转换矩阵
| 当前状态 | 可转换到 | 触发条件 | 动作 |
|----------|----------|----------|------|
| PAPER_IDLE | PAPER_PRE_CHECK | M0命令或按钮按下 | 开始检查当前状态 |
| PAPER_IDLE | PAPER_EJECTING | 检测到有纸 | 执行出纸流程 |
| PAPER_PRE_CHECK | PAPER_EJECTING | 传感器检测到纸张 | 切换到出纸状态 |
| PAPER_PRE_CHECK | PAPER_FEEDING | 传感器无纸张 | 切换到进纸状态 |
| PAPER_EJECTING | PAPER_COMPLETE | 出纸完成 | 系统就绪 |
| PAPER_FEEDING | PAPER_UNCLAMP_FEED | 传感器检测到纸张 | 松开夹紧器进纸 |
| PAPER_UNCLAMP_FEED | PAPER_CLAMP_FEED | 进纸3.5cm完成 | 开始夹紧进纸 |
| PAPER_CLAMP_FEED | PAPER_REPOSITION | 纸张脱离传感器 | 重新定位 |
| PAPER_REPOSITION | PAPER_COMPLETE | 位置校准完成 | 换纸完成 |
| 任意状态 | PAPER_ERROR | 超时或错误 | 进入错误处理 |
| PAPER_ERROR | PAPER_IDLE | 错误处理完成 | 返回空闲状态 |

#### 状态机实现原理

##### 1. 事件驱动模型
```cpp
// 状态机主循环
void paper_change_state_machine() {
    // 获取当前传感器状态
    int paper_detected = digitalRead(PAPER_SENSOR_PIN);
    uint32_t current_time = millis();
    
    // 检查状态转换条件
    switch (paper_ctrl.state) {
        case PAPER_IDLE:
            if (check_paper_change_trigger()) {
                transition_to_state(PAPER_PRE_CHECK);
            }
            break;
            
        case PAPER_PRE_CHECK:
            if (paper_detected) {
                transition_to_state(PAPER_EJECTING);
            } else {
                transition_to_state(PAPER_FEEDING);
            }
            break;
            
        // ... 其他状态处理
    }
    
    // 检查超时条件
    check_timeout_conditions();
}
```

##### 2. 状态转换函数
```cpp
void transition_to_state(paper_state_t new_state) {
    // 记录状态变化
    log_state_change(paper_ctrl.state, new_state);
    
    // 执行退出当前状态的动作
    exit_current_state();
    
    // 更新状态
    paper_ctrl.state = new_state;
    paper_ctrl.start_time = millis();
    
    // 执行进入新状态的动作
    enter_new_state();
}
```

##### 3. 动作执行机制
```cpp
// 进入状态时的动作
void enter_new_state() {
    switch (paper_ctrl.state) {
        case PAPER_FEEDING:
            // 启动进纸电机
            start_feed_motor();
            break;
            
        case PAPER_CLAMP_FEED:
            // 启动夹紧电机
            start_clamp_motor();
            // 同时启动进纸和面板电机
            start_feed_and_panel_motors();
            break;
            
        // ... 其他状态的进入动作
    }
}

// 退出状态时的动作
void exit_current_state() {
    switch (paper_ctrl.previous_state) {
        case PAPER_FEEDING:
            // 停止进纸电机
            stop_feed_motor();
            break;
            
        // ... 其他状态的退出动作
    }
}
```

### 状态机优势

#### 1. 可靠性保证
- **确定性**：每个状态都有明确的进入和退出条件
- **完整性**：覆盖所有可能的运行情况
- **安全性**：异常状态自动进入错误处理
- **可追溯**：完整的状态转换历史记录

#### 2. 可维护性
- **模块化**：每个状态的逻辑独立
- **可扩展**：容易添加新状态或转换条件
- **可调试**：状态转换过程完全可观测
- **可测试**：每个状态可以独立测试

#### 3. 实时性保证
- **非阻塞**：状态检查不阻塞主循环
- **响应性**：快速响应传感器和命令输入
- **时效性**：超时保护确保系统不卡死
- **优先级**：紧急状态优先处理

#### 通信协议时序
1. **锁存阶段**：拉低STCP，准备数据移入
2. **移位阶段**：8个时钟周期，从MSB到LSB逐位移入数据
3. **数据建立**：每个时钟上升沿前数据线必须稳定1μs
4. **锁存输出**：拉高STCP，8位数据同时并行输出到Q0-Q7
5. **硬件稳定**：等待10μs确保输出信号稳定

#### HR4988电流设置与计算（QFN28标准版）
HR4988使用固定关闭时间PWM电流控制，通过REF引脚设置参考电压，配合检流电阻进行电流控制。

**电流控制原理：**
```cpp
// HR4988电流控制公式
// V_sense = I_trip × R_sense
// I_trip = VREF / (3 × R_sense)  // 简化计算公式
// 更精确的公式考虑放大器增益和内部参考电压

// 常用电流设置（假设R_sense = 0.1Ω）
// 要设置1.0A电流：VREF = I_trip × 3 × R_sense = 1.0A × 3 × 0.1Ω = 0.3V
// 要设置0.8A电流：VREF = 0.8A × 3 × 0.1Ω = 0.24V

// VREF电压分压计算
// VREF = VDD × (R2 / (R1 + R2))
// 其中VDD = 3.3V（或5V），R1、R2是分压电阻

// 实际电流设置参数（参考值）
#define X_AXIS_CURRENT_MA         1000    // X轴电流：1000mA
#define Y_AXIS_CURRENT_MA         1000    // Y轴电流：1000mA（双电机总电流）
#define Z_AXIS_CURRENT_MA         800     // Z轴电流：800mA（抬笔负载较小）
#define PAPER_MOTOR_CURRENT_MA    1000    // 换纸电机电流：1000mA

// 对应的VREF电压设置（R_sense = 0.1Ω）
#define X_AXIS_VREF_MV           300     // X轴VREF：300mV (1.0A)
#define Y_AXIS_VREF_MV           300     // Y轴VREF：300mV (1.0A)
#define Z_AXIS_VREF_MV           240     // Z轴VREF：240mV (0.8A)
#define PAPER_MOTOR_VREF_MV      300     // 换纸电机VREF：300mV (1.0A)

// 分压电阻计算（VDD = 3.3V，R_sense = 0.1Ω）
// 1.0A：R1=10kΩ, R2=1kΩ (VREF≈300mV)
// 0.8A：R1=10kΩ, R2=820Ω (VREF≈240mV)
```

| 元件类型 | 推荐值 | 连接位置 | 作用 |
|----------|--------|----------|------|
| 检流电阻R_sense | 0.1Ω | SENSE1/2到GND | 电流检测 |
| 电荷泵飞电容 | 0.1μF | CP1 ↔ CP2 | 电荷泵储能 |

**电流配置注意事项：**
- HR4988最大输出电流：2.5A（需良好散热）
- 推荐工作电流：≤2.0A（保证稳定性）

```cpp
    hc595_write(hc595_data);
    delayMicroseconds(MOTOR_DIRECTION_SETTLE_US);
    
    // 执行步进脉冲
    for (uint32_t i = 0; i < steps; i++) {
        hc595_data |= get_step_bit(motor);  // 步进脉冲高电平

        }
    }
```

```cpp
    static uint8_t buffer_index = 0;
    static uint8_t true_count = 0;
    
    // 读取原始传感器值

        true_count--;
```

### 故障排除指南

#### 1. 常见问题诊断

**问题：换纸不工作**
- 检查AUTO_PAPER_CHANGE_ENABLE是否为1
- 验证HC595连接是否正确
- 查看串口是否有错误信息

**问题：引脚冲突**
- 查看启动日志中的引脚冲突报告
- 确认GPIO2未被同时用于X_STEP_PIN和HC595_LATCH_PIN
- 检查GPIO4（STEPPERS_DISABLE_PIN）配置

**问题：电机不动作**
- 验证HC595输出位映射是否正确
- 检查电机驱动器电源
- 确认使能信号正确

**问题：传感器检测异常**
- 验证GPIO34（PAPER_SENSOR_PIN）连接
- 检查传感器供电和信号电平
- 测试GPIO35（PAPER_BUTTON_PIN）按钮功能

**问题：系统无响应或死锁**
- 尝试 `$PaperReset` 重置换纸系统
- 使用 `$X` 清除GRBL缓冲区
- 如果完全无响应，使用 `$Ctrl+X` 软重启
- 最后手段：物理重启硬件（断电重启）

**问题：换纸状态机卡在某个状态**
- 使用 `$PaperState=RESET` 重置状态机
- 检查 `$PaperState=IDLE` 是否能返回空闲状态
- 查看调试日志确定卡住的具体原因
- 必要时使用 `$HardReset` 硬件复位

**问题：HC595输出异常**
- 执行 `$PaperInit` 重新初始化HC595
- 使用 `$PaperMotorsReset` 停止所有电机
- 检查HC595供电和数据连接
- 验证引脚配置是否正确

**问题：重启后配置丢失**
- 确认EEPROM/Flash存储正常
- 检查 `$Settings` 命令查看当前参数
- 使用 `$$` 查看所有设置参数
- 必要时重新配置并保存到存储器

```cpp
// 单步控制进纸电机（示例代码）
void paper_feed_step_forward() {
    uint8_t data = 0;
    data |= (1 << BIT_FEED_MOTOR_DIR);    // 设置方向为正转
    hc595_write(data);                    // 输出方向
    delayMicroseconds(PAPER_FEED_INTERVAL_US/2);
    
    data |= (1 << BIT_FEED_MOTOR_STEP);   // 步进脉冲高电平
    hc595_write(data);                    // 输出脉冲
    delayMicroseconds(DEFAULT_STEP_PULSE_MICROSECONDS);
    
    data &= ~(1 << BIT_FEED_MOTOR_STEP);  // 步进脉冲低电平
    hc595_write(data);                    // 完成脉冲
    delayMicroseconds(PAPER_FEED_INTERVAL_US/2);
}

// 连续步进控制
void paper_feed_steps(uint32_t steps, bool forward) {
    uint8_t data = 0;
    if (!forward) {
        data |= (1 << BIT_FEED_MOTOR_DIR); // 反转方向
    } else {
        data &= ~(1 << BIT_FEED_MOTOR_DIR); // 正转方向
    }
    
    for (uint32_t i = 0; i < steps; i++) {
        hc595_write(data);
        delayMicroseconds(PAPER_FEED_INTERVAL_US/2);
        
        data |= (1 << BIT_FEED_MOTOR_STEP);
        hc595_write(data);
        delayMicroseconds(DEFAULT_STEP_PULSE_MICROSECONDS);
        
        data &= ~(1 << BIT_FEED_MOTOR_STEP);
        hc595_write(data);
        delayMicroseconds(PAPER_FEED_INTERVAL_US/2);
    }
}
```

**调试注意事项：**
1. 修改配置后需要重新编译和上传
2. 调试模式会增加串口输出，可能影响实时性能
3. 硬件测试时注意安全，避免电机碰撞
4. 保存调试日志以便分析问题根因
5. 定期检查硬件连接的可靠性

**重启操作注意事项：**
6. 优先使用软重启（$PaperReset、$X），避免频繁硬重启
7. 重启前确保所有电机已停止，避免机械损伤
8. 系统重启后检查配置参数是否保持正确
9. 频繁重启可能影响硬件寿命，谨慎使用
10. 保存重要数据后再执行重启操作
11. 看门狗重启功能需要编译时启用才有效
12. 硬件重启作为最后手段，仅在系统完全无响应时使用

**单步调试注意事项：**
13. 单步指令会绕过安全检查，使用时确保机械空间充足
14. 单步电机控制时注意步进间隔，避免过快导致失步
15. 使用强制状态指令前，了解当前状态避免冲突
16. 传感器强制设置仅用于测试，正常流程中不要使用
17. 单步调试时会输出大量日志，注意串口缓冲区大小
18. 长时间单步操作可能导致过热，注意散热
19. 调试完成后重置系统到正常状态，清除调试设置
20. 单步模式下的位置计算可能不准确，需要重新校准

**串口调试注意事项：**
21. 串口波特率必须与GRBL设置一致（默认115200）
22. 实时状态报告会增加串口负载，必要时关闭
23. 数据记录功能会消耗内存，定期清理或导出
24. 串口缓冲区溢出时系统可能响应变慢
25. 回显模式仅用于测试，正常使用时关闭
26. 长时间串口连接可能导致ESP32过热
27. 复杂指令处理期间避免发送新命令
28. 数据记录功能启用后注意Flash写入次数限制
29. 串口通信异常时检查USB连接和驱动
30. 实时监控模式可能影响电机运动精度

```cpp
bool auto_sensor_calibration() {
    grbl_sendf(CLIENT_ALL, "[PAPER-INFO] 开始传感器校准...\n");
    
    // 步骤1：无纸状态校准
    grbl_sendf(CLIENT_ALL, "[PAPER-INFO] 请移除所有纸张，按继续键...\n");
    wait_for_user_confirmation();
    
    uint16_t no_paper_samples[100];
    for (int i = 0; i < 100; i++) {
        no_paper_samples[i] = analogRead(PAPER_SENSOR_ANALOG_PIN);
        delay(10);
    }
    uint16_t no_paper_threshold = calculate_average(no_paper_samples, 100);
    
    // 步骤2：有纸状态校准
    grbl_sendf(CLIENT_ALL, "[PAPER-INFO] 请放置纸张，按继续键...\n");
    wait_for_user_confirmation();
    
    uint16_t with_paper_samples[100];
    for (int i = 0; i < 100; i++) {
        with_paper_samples[i] = analogRead(PAPER_SENSOR_ANALOG_PIN);
        delay(10);
    }
    uint16_t with_paper_threshold = calculate_average(with_paper_samples, 100);
    
    // 步骤3：计算最优阈值
    uint16_t optimal_threshold = (no_paper_threshold + with_paper_threshold) / 2;
    
    // 步骤4：验证校准结果
    bool calibration_success = test_sensor_threshold(optimal_threshold);
    
    if (calibration_success) {
        save_sensor_calibration(no_paper_threshold, with_paper_threshold, 
                             optimal_threshold);
        grbl_sendf(CLIENT_ALL, "[PAPER-INFO] 传感器校准完成\n");
        return true;
    } else {
        grbl_sendf(CLIENT_ALL, "[PAPER-ERROR] 传感器校准失败\n");
```

```cpp
// 精确步进校准
bool motor_step_calibration() {
    grbl_sendf(CLIENT_ALL, "[PAPER-INFO] 开始电机步进校准...\n");
    
    for (int motor = 0; motor < 3; motor++) {
        // 校准每个电机的实际步进精度
        bool success = calibrate_single_motor(motor);
        
        if (!success) {
            grbl_sendf(CLIENT_ALL, "[PAPER-ERROR] 电机%d校准失败\n", motor);
            return false;
        }
    }
    
    // 测试协调运动精度
    bool coordination_success = test_motor_coordination();
    
    if (coordination_success) {
        save_motor_calibration_data();
        grbl_sendf(CLIENT_ALL, "[PAPER-INFO] 电机校准完成\n");
        return true;
    } else {
        grbl_sendf(CLIENT_ALL, "[PAPER-ERROR] 电机协调测试失败\n");
        return false;
    }
}

bool calibrate_single_motor(motor_type_t motor) {
    // 使用激光尺或编码器进行精确校准
    const uint32_t target_distance = 100.0;  // 目标：100mm
    const uint32_t theoretical_steps = target_distance * get_steps_per_mm(motor);
    
    grbl_sendf(CLIENT_ALL, "[PAPER-INFO] 电机%d：%d步理论距离%.1fmm\n", 
               motor, theoretical_steps, target_distance);
    
    // 执行运动
    move_motor(motor, theoretical_steps, MOTOR_FORWARD);
    
    // 等待用户输入实际距离
    grbl_sendf(CLIENT_ALL, "[PAPER-INFO] 请输入实际移动距离（mm）：\n");
    float actual_distance = wait_for_float_input();
    
    // 计算校准系数
    float calibration_factor = actual_distance / target_distance;
    float corrected_steps_per_mm = get_steps_per_mm(motor) / calibration_factor;
    
    // 验证校准精度
    if (fabs(calibration_factor - 1.0) < 0.05) {  // 5%误差内
        save_motor_calibration(motor, corrected_steps_per_mm);
        return true;
    } else {
        grbl_sendf(CLIENT_ALL, "[PAPER-WARN] 校准误差%.1f%%，请重新校准\n", 
                   fabs(calibration_factor - 1.0) * 100);
        return false;
    }
}
```
