## 🏗️ 系统架构概述

### 核心组件

换纸系统采用**状态机设计模式**，由以下核心组件构成：

```
┌─────────────────────────────────────────┐
│        换纸系统主循环 (paper_change_update) │
│  - 传感器状态采样                        │
│  - 状态机逻辑更新                        │
│  - 按钮状态检查                          │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│      状态机调度器 (handle_state_machine)  │
│  - 根据当前状态调用相应处理函数           │
│  - 状态转换验证                          │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│      状态处理函数 (handle_xxx_state)     │
│  - 执行具体业务逻辑                      │
│  - 控制电机运动                          │
│  - 检测传感器状态                        │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│      硬件控制层 (PaperChangeHardware)   │
│  - HC595移位寄存器控制                  │
│  - 电机步进控制                          │
│  - 传感器读取                            │
└─────────────────────────────────────────┘
```

## 🔄 状态机设计

### 状态定义

系统定义了**10个状态**：

| 状态 | 说明 | 功能 |
|------|------|------|
| `PAPER_IDLE` | 空闲状态 | 系统待命，等待触发 |
| `PAPER_PRE_CHECK` | 预检状态 | 检测面板是否有纸 |
| `PAPER_EJECTING` | 出纸状态 | 送出已写完的纸张 |
| `PAPER_FEEDING` | 进纸状态 | 送入新纸张 |
| `PAPER_UNCLAMP_FEED` | 松开进纸 | 松开夹具并送纸 |
| `PAPER_CLAMP_FEED` | 夹紧进纸 | 夹紧纸张 |
| `PAPER_FULL_FEED` | 完整进给 | A4行程进给 |
| `PAPER_REPOSITION` | 重新定位 | 精确定位到写字位置 |
| `PAPER_COMPLETE` | 完成状态 | 换纸完成 |
| `PAPER_ERROR` | 错误状态 | 系统异常处理 |

### 状态转换矩阵

系统使用**状态转换矩阵**确保状态转换的合法性：

```cpp
// 示例：合法的状态转换
IDLE → PRE_CHECK → EJECTING → FEEDING → UNCLAMP_FEED → 
CLAMP_FEED → FULL_FEED → REPOSITION → COMPLETE → IDLE
```

**关键特性**：
- 每个状态只能转换到预定义的下一个状态
- 非法转换会被拒绝，自动进入ERROR状态
- 所有状态都可以转换到ERROR状态（安全机制）

### 状态转换函数 (`enter_state`)

状态转换的完整流程：

```cpp
void enter_state(paper_change_state_t new_state) {
    // 1. 验证状态转换合法性
    if (!validate_state_transition(new_state)) {
        enter_state(PAPER_ERROR);  // 非法转换 → 错误状态
        return;
    }
    
    // 2. 处理错误状态恢复
    handle_error_state_recovery(old_state, new_state);
    
    // 3. 重置静态标志
    reset_static_flags(new_state);
    
    // 4. 更新控制结构
    update_control_structure(new_state);  // 重置step_counter=0, 更新state_timer
    
    // 5. 完成状态转换
    complete_state_transition(old_state, new_state);  // 初始化电机时序，记录日志
}
```

## 🔀 完整流程详解

### 主循环 (`paper_change_update`)

这是系统的**心脏**，在主循环中持续调用：

```cpp
void paper_change_update() {
    // 步骤1：传感器状态更新和跳变检测
    ctrl->last_paper_sensor_state = ctrl->paper_sensor_state;  // 保存上一状态
    ctrl->paper_sensor_state = read_paper_sensor_debounced(); // 读取当前状态
    
    // 步骤2：状态机逻辑更新
    paper_state_machine_update();  // 根据当前状态调用相应处理函数
    
    // 步骤3：HR4988 VREF控制更新（动态电流控制）
    update_hr4988_vref();
    
    // 步骤4：用户交互检查
    check_button_press();  // 检查一键换纸按钮
}
```

### 流程A：面板上有纸张（需要先出纸）

#### 1. IDLE（空闲）→ PRE_CHECK（预检）

**触发条件**：
- M0命令
- 一键换纸按钮按下

**PRE_CHECK状态逻辑**：
```cpp
void handle_pre_check_state() {
    // 步骤1：面板电机反转3mm，检测面板是否有纸
    if (ctrl->step_counter < PAPER_PRE_CHECK_STEPS) {  // 240步（3mm × 80 steps/mm）
        if (nonblocking_panel_step(false)) {  // 反转
            ctrl->step_counter++;
        }
    } 
    // 步骤2：根据传感器状态决定后续流程
    else {
        if (ctrl->paper_sensor_state) {
            // 检测到纸张 → 进入EJECTING
            enter_state(PAPER_EJECTING);
        } else {
            // 未检测到纸张 → 进入FEEDING
            enter_state(PAPER_FEEDING);
        }
    }
}
```

**关键点**：
- 反转3mm清除可能的纸张遮挡
- 传感器状态决定后续流程分支

---

#### 2. PRE_CHECK → EJECTING（出纸）

**EJECTING状态逻辑**：
```cpp
void handle_ejecting_state() {
    // 完全出纸 - 面板电机快速正转
    // 距离：A4(297mm) + 预留(40mm) = 337mm
    // 步数：26960步（337mm × 80 steps/mm）
    if (ctrl->step_counter < (PAPER_EJECT_STEPS + PAPER_3TO5CM_STEPS)) {
        handle_full_ejection(ctrl, nullptr);
    } else {
        // 出纸完成 → 进入FEEDING
        STOP_ALL_MOTORS();
        enter_state(PAPER_FEEDING);
    }
}
```

**关键点**：
- **直接完全出纸**，无需边缘检测（已简化）
- 速度：2kHz（500μs间隔）
- 出纸完成后自动进入FEEDING状态

---

#### 3. EJECTING → FEEDING（进纸）

**FEEDING状态逻辑**：
```cpp
void handle_feeding_state() {
    // 步骤1：等待纸张前边缘检测
    if (ctrl->paper_sensor_state && !ctrl->last_paper_sensor_state) {
        // 检测到前边缘（HIGH→LOW跳变）
        LOG_MSG("Paper front edge detected");
        STOP_ALL_MOTORS();  // 立即停止，精确定位
        enter_state(PAPER_UNCLAMP_FEED);
    } 
    // 步骤2：超时保护
    else if (ctrl->step_counter > PAPER_FEED_TIMEOUT_STEPS) {  // 2900步
        handle_error(ERROR_SENSOR_TIMEOUT, "Feed timeout", PAPER_FEEDING);
    } 
    // 步骤3：持续进纸
    else {
        if (nonblocking_feed_step(true)) {  // 送纸器电机正转
            ctrl->step_counter++;
        }
    }
}
```

**关键点**：
- 送纸器电机**只正转，永不反转**（机械设计决定）
- 等待传感器检测到纸张前边缘（HIGH→LOW跳变）
- 超时保护：2900步（约36.25mm）未检测到则报错

---

#### 4. FEEDING → UNCLAMP_FEED（松开进纸）

**UNCLAMP_FEED状态逻辑**：
```cpp
void handle_unclamp_feed_state() {
    // 步骤5a：夹紧电机正转1.5cm松开夹具（1200步）
    if (ctrl->step_counter < PAPER_1_5CM_STEPS) {
        if (nonblocking_clamp_step(true)) {  // 正转松开
            ctrl->step_counter++;
        }
    } 
    // 步骤5b：送纸器电机正转4cm送纸（3200步）
    else if (ctrl->step_counter < PAPER_1_5CM_STEPS + PAPER_4CM_STEPS) {
        if (nonblocking_feed_step(true)) {  // 送纸器电机正转
            ctrl->step_counter++;
        }
    } 
    // 完成 → 进入CLAMP_FEED
    else {
        enter_state(PAPER_CLAMP_FEED);
    }
}
```

**关键点**：
- **顺序执行**：先完成5a（松开），再执行5b（送纸）
- 拾落电机松开后，面板电机无法带动纸张，**必须使用送纸器电机**
- 总步数：4400步（1200 + 3200）

---

#### 5. UNCLAMP_FEED → CLAMP_FEED（夹紧进纸）

**CLAMP_FEED状态逻辑**：
```cpp
void handle_clamp_feed_state() {
    // 夹紧电机反转1.5cm到初始位置夹紧纸张（1200步）
    if (ctrl->step_counter < PAPER_1_5CM_STEPS) {
        if (nonblocking_clamp_step(false)) {  // 反转夹紧
            ctrl->step_counter++;
        }
    } else {
        // 夹紧完成 → 进入FULL_FEED
        enter_state(PAPER_FULL_FEED);
    }
}
```

**关键点**：
- 夹紧电机反转回到初始位置
- 夹紧完成后纸张被固定

---

#### 6. CLAMP_FEED → FULL_FEED（完整进给）

**FULL_FEED状态逻辑**（**两阶段设计**）：

```cpp
void handle_full_feed_state() {
    // 阶段1：同步运行阶段 - 两个电机同步运行，直到纸张脱离传感器
    if (!reverse_complete_val) {
        // 设置两个电机同步运行，相同步进间隔（1000μs）
        feed_timing->step_interval = 1000;
        panel_timing->step_interval = 1000;
        
        // 检查完成条件
        if ((ctrl->step_counter >= PAPER_EJECT_STEPS && !ctrl->paper_sensor_state) ||
            (!ctrl->paper_sensor_state && ctrl->step_counter >= min_feed_steps)) {
            // 纸张脱离传感器 → 停止进纸器电机，准备面板电机单独运行
            feed_timing->last_step_time = 0;  // 禁用进纸器电机
            ctrl->step_counter = 0;  // 重置计数器
            *panel_alone_complete = true;  // 标记进入阶段2
        } else {
            // 同步步进两个电机
            motor_step_control(BIT_FEED_MOTOR_STEP, ...);
            motor_step_control(BIT_PANEL_MOTOR_STEP, ...);
            ctrl->step_counter++;
        }
    }
    // 阶段2：面板电机单独运行阶段 - 面板电机单独运行3cm
    else if (reverse_complete_val && ctrl->step_counter < PAPER_3CM_STEPS) {
        // 面板电机单独运行，进纸器电机已停止
        if (nonblocking_panel_step(true)) {
            ctrl->step_counter++;
            
            if (ctrl->step_counter >= PAPER_3CM_STEPS) {  // 2400步（30mm）
                // 完成 → 进入REPOSITION
                STOP_ALL_MOTORS();
                enter_state(PAPER_REPOSITION);
            }
        }
    }
}
```

**关键点**：
- **阶段1**：送纸器电机 + 面板电机**同步运行**（相同间隔1000μs）
- **阶段2**：纸张脱离传感器后，进纸器电机停止，**面板电机单独运行3cm**
- 使用`reverse_complete`标志（复用）跟踪阶段切换

---

#### 7. FULL_FEED → REPOSITION（重新定位）

**REPOSITION状态逻辑**（**两阶段设计**）：

```cpp
void handle_reposition_state() {
    // 步骤8a：反转寻找边缘
    if (!reverse_complete_val && ctrl->step_counter < PAPER_MAX_REVERSE_STEPS) {
        if (nonblocking_panel_step(false)) {  // 反转
            ctrl->step_counter++;
            
            // 检测纸张前边缘：从无纸(false)到有纸(true)的跳变
            if (ctrl->paper_sensor_state && !ctrl->last_paper_sensor_state) {
                LOG_MSG("Paper edge detected in reverse");
                ctrl->step_counter = 0;  // 重置计数器
                reverse_complete_val = true;  // 标记进入步骤8b
            }
        }
    } 
    // 步骤8b：正转3cm定位（关键步骤）
    else if (reverse_complete_val && ctrl->step_counter < PAPER_3CM_STEPS) {
        if (nonblocking_panel_step(true)) {  // 正转
            ctrl->step_counter++;
            
            if (ctrl->step_counter >= PAPER_3CM_STEPS) {  // 2400步（30mm）
                // 定位完成 → 进入COMPLETE
                enter_state(PAPER_COMPLETE);
            }
        }
    }
}
```

**关键点**：
- **步骤8a**：面板电机反转，直到传感器再次检测到纸张前边缘
- **步骤8b**：面板电机正转3cm，精确定位到写字位置（**关键步骤**）
- 速度：2kHz（500μs间隔，精确控制）

---

#### 8. REPOSITION → COMPLETE（完成）

**COMPLETE状态逻辑**：

```cpp
void handle_complete_state() {

}
```

## ⚙️ 核心机制

### 1. 非阻塞电机控制

所有电机控制都是**非阻塞**的，使用时间间隔控制：

```cpp
bool nonblocking_panel_step(bool forward) {
    motor_timing_t* timing = get_panel_motor_timing();
    uint32_t current_time = millis();
    
    // 检查是否到了执行步进的时间
    uint32_t elapsed = (current_time >= timing->last_step_time) ? 
                      (current_time - timing->last_step_time) :
                      (UINT32_MAX - timing->last_step_time + current_time + 1);
    
    if (elapsed >= timing->step_interval) {
        // 执行步进
        motor_step_control(BIT_PANEL_MOTOR_STEP, BIT_PANEL_MOTOR_DIR, forward, timing->step_interval);
        timing->last_step_time = current_time;
        return true;  // 执行了步进
    }
    return false;  // 未到时间，未执行步进
}
```

**关键特性**：
- 使用`millis()`时间戳控制步进间隔
- 安全处理`millis()`溢出（49天后）
- 每次调用只执行一次步进，不阻塞

### 2. 传感器跳变检测

传感器状态变化检测使用**边沿检测**：

```cpp
// 在主循环中更新传感器状态
ctrl->last_paper_sensor_state = ctrl->paper_sensor_state;  // 保存上一状态
ctrl->paper_sensor_state = read_paper_sensor_debounced();   // 读取当前状态

// 在状态处理函数中检测跳变
if (ctrl->paper_sensor_state && !ctrl->last_paper_sensor_state) {
    // HIGH→LOW跳变（检测到纸张前边缘）
    LOG_MSG("Paper front edge detected");
}
```

**传感器逻辑**：
- 硬件：GPIO34，低电平有效（检测到纸张时为LOW）
- 软件：`paper_sensor_state=true`表示检测到纸张
- 跳变检测：`!last_state && current_state` = 检测到前边缘

### 3. 静态标志管理

系统使用**静态标志**在状态之间传递信息：

```cpp
bool positioning_init;      // 位置初始化标志

void get_static_flags(bool* pos_init, bool* eject_detected, bool* reverse_complete);

```

**使用场景**：
- `reverse_complete`：在FULL_FEED和REPOSITION状态中复用，用于阶段切换

### 4. 同步电机控制

FULL_FEED状态中，两个电机需要**真正同步运行**：

```cpp
// 只有当两个电机都到了应该步进的时间时，才同步执行步进
uint32_t feed_elapsed = ...;  // 计算进纸器电机经过的时间
uint32_t panel_elapsed = ...; // 计算面板电机经过的时间

if (feed_elapsed >= feed_timing->step_interval && 
    panel_elapsed >= panel_timing->step_interval) {
    // 同步步进两个电机
    motor_step_control(BIT_FEED_MOTOR_STEP, ...);
    motor_step_control(BIT_PANEL_MOTOR_STEP, ...);
    
    // 更新时序和计数器
    feed_timing->last_step_time = current_time;
    panel_timing->last_step_time = current_time;
    ctrl->step_counter++;  // 每次两个电机同步步进时，计数器加1
}
```

**关键点**：
- 两个电机使用**相同的步进间隔**（1000μs）
- 只有当**两个电机都到时间**时才执行步进
- 确保两个电机真正同步

## 🛡️ 安全保护机制

### 1. 状态超时保护

每个状态都有**最大步数限制**和**时间超时**：

```cpp
// 安全检查宏
#define CHECK_STATE_SAFETY(max_steps, state_name) \
    do { \
        if (!runtime_safety_check(ctrl->step_counter, (max_steps), (state_name))) { \
            return; \
        } \
    } while(0)

// 使用示例
CHECK_STATE_SAFETY(PAPER_PRE_CHECK_STEPS + 50, "PRE_CHECK");
```

**保护机制**：
- **步数超时**：超过最大步数 → 进入ERROR状态
- **时间超时**：超过30秒（`PAPER_STATE_TIMEOUT_MS`） → 进入ERROR状态
- **进纸超时**：2900步（约36.25mm）未检测到纸张 → 报错

---

### 2. 状态转换验证

所有状态转换都经过**合法性验证**：

```cpp
bool is_valid_transition(paper_change_state_t from_state, paper_change_state_t to_state) {
    // 使用状态转换矩阵验证
    return state_transition_matrix[from_state][to_state];
}
```

**保护机制**：
- 非法转换会被拒绝
- 自动进入ERROR状态
- 防止状态机混乱

代码中包含了**边界情况处理**，防止状态卡住：

```cpp
// 示例：EJECTING状态的边界情况
else {
    LOG_WARNING_F("EJECTING state: counter exceeded, forcing transition to FEEDING");
    STOP_ALL_MOTORS();
    enter_state(PAPER_FEEDING);
}
```

```cpp
// 位映射定义
#define BIT_PAPER_CLAMP_DIR     2     // 夹纸电机方向控制位
```

**控制方式**：
- 8位输出，控制3个电机的方向和步进
- 使用`hc595_update()`函数更新输出
- 非阻塞控制，不占用CPU时间

### 电机参数配置

所有电机使用**80 steps/mm**配置：

```cpp
#define DEFAULT_FEED_STEPS_PER_MM   80.0  // 进纸电机
#define DEFAULT_PANEL_STEPS_PER_MM  80.0  // 面板电机
#define DEFAULT_CLAMP_STEPS_PER_MM  80.0  // 夹紧电机
```

**计算依据**：
- 步进电机：200步/圈（1.8°）
- 微步配置：1/16
- 同步带：GT2（2mm齿距）
- 同步带轮：20齿
- 计算：`(200 × 16) ÷ (2 × 20) = 80 steps/mm`
