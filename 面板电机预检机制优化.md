# 面板电机预检机制优化

## 🎯 优化背景

### 用户建议的核心思路
> "在面板电机拉出已经写好的纸时，是不是倒转一下，来确定面板上有无纸张，没纸的话，就可以使用进纸器进纸了，有纸的话就用面板电机出纸"

这个建议解决了**传感器误判**的关键问题，避免无效的进纸操作。

---

## 🔄 新增的预检状态机

### 原始状态机问题
```
OLD: IDLE → EJECTING → FEEDING → ...
     ↑      直接假设面板有纸
```

### 优化后的状态机
```
NEW: IDLE → PRE_CHECK → EJECTING/FEEDING → ...
     ↑      先检测再决策     ↑     根据检测结果分支
```

---

## 📋 具体实现方案

### 1. **新增状态定义**
```cpp
typedef enum {
    PAPER_IDLE,              // 空闲状态
    PAPER_PRE_CHECK,         // 预检状态 - 面板电机倒转检测是否有纸
    PAPER_EJECTING,          // 出纸状态 - 面板电机送出写完的纸
    PAPER_FEEDING,           // 进纸状态 - 进纸器送入新纸
    // ... 其他状态保持不变
} paper_change_state_t;
```

### 2. **预检状态逻辑**
```cpp
case PAPER_PRE_CHECK:
    // Pre-check state - 面板电机倒转检测是否有纸
    if (paper_ctrl.step_counter < PAPER_PRE_CHECK_STEPS) {  // 倒转50步(0.625mm)
        generate_motor_step(BIT_PANEL_MOTOR_STEP, BIT_PANEL_MOTOR_DIR, false); // 倒转
        paper_ctrl.step_counter++;
        delayMicroseconds(PAPER_PRE_CHECK_INTERVAL_US);
    } else {
        // 检测倒转后的传感器状态
        if (paper_ctrl.paper_sensor_state) {
            // 传感器仍检测到纸 → 面板上有纸，执行出纸
            grbl_sendf(CLIENT_ALL, "[MSG: Paper detected on panel, starting ejection]\r\n");
            enter_state(PAPER_EJECTING);
        } else {
            // 传感器未检测到纸 → 面板无纸，直接进纸
            grbl_sendf(CLIENT_ALL, "[MSG: No paper on panel, starting feeding]\r\n");
            enter_state(PAPER_FEEDING);
        }
    }
    break;
```

## 📊 优化效果分析

### 1. **解决传感器误判问题**

#### 原始问题
```
场景1: 面板实际无纸，但传感器误报为有纸
结果: 无效出纸操作，浪费时间

场景2: 面板实际有纸，但传感器误报为无纸  
结果: 错误进纸操作，可能卡纸
```

#### 优化后解决方案
```
预检机制: 倒转0.625mm确认传感器状态
- 如果倒转后仍有信号 → 确认有纸 → 执行出纸
- 如果倒转后信号消失 → 确认无纸 → 执行进纸
```

### 2. **提高系统可靠性**

| 场景 | 原始方案 | 优化方案 | 可靠性提升 |
|------|----------|----------|------------|
| 传感器正常 | 正确执行 | 正确执行 | 无变化 |
| 传感器误报高电平 | 错误出纸 | 预检确认 | ✅ 避免误操作 |
| 传感器误报低电平 | 错误进纸 | 预检确认 | ✅ 避免卡纸 |
| 纸张边缘模糊 | 不稳定 | 倒转确认 | ✅ 状态明确 |

### 3. **性能优化**

#### 预检时间成本
```
预检倒转: 50步 × 1ms = 50ms
检测判断: 瞬时
总时间成本: 50ms (几乎可忽略)
```

#### 避免的浪费
```
无效出纸避免: 23800步 × 1ms = 23.8秒
无效进纸避免: 500步超时 × 2ms = 1秒 + 人工干预
净收益: 节省1-24秒时间
```

## 🔧 参数配置

### 新增配置参数
```cpp
// Paper Pre-Check Parameters  
#define PAPER_PRE_CHECK_STEPS       50        // steps - 预检倒转步数(约0.625mm)
#define PAPER_PRE_CHECK_INTERVAL_US  1000      // μs - 预检倒转间隔
```

### 参数设计考虑
- **50步距离**: 0.625mm，足够检测但又不会过度倒转
- **1ms间隔**: 与出纸速度一致，保持电机平滑运行
- **可调参数**: 便于根据实际硬件调整

## 💡 创新亮点

### 1. **主动式检测**
- **被动检测**: 依赖传感器静态状态
- **主动检测**: 倒转动态确认传感器响应

### 2. **零成本改进**
- **硬件改动**: 无需任何硬件修改
- **软件优化**: 纯逻辑增强
- **时间成本**: 仅增加50ms预检时间

### 3. **智能化决策**
- **条件分支**: 根据预检结果智能选择操作
- **避免误判**: 大幅降低传感器误判影响
- **提高效率**: 避免无效的进纸/出纸操作
