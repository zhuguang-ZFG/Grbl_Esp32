## 🎯 系统概述

### 核心功能

自动完成以下任务：
1. **检测纸张状态** - 判断面板上是否有纸
2. **送出旧纸张** - 将已写完的纸张完全送出
3. **送入新纸张** - 从纸盒中送入新纸张
4. **精确定位** - 将纸张定位到写字位置

### 硬件组成

- **3个步进电机**：
  - 面板电机 - 控制纸张在面板上的移动（正转/反转）
  - 进纸电机 - 从纸盒送入新纸张（**只正转，永不反转**）
  - 夹紧电机（拾落电机）- 控制夹具松开/夹紧（正转松开，反转夹紧）

- **传感器**：
  - 纸张传感器 - 检测纸张位置（GPIO34，低电平有效）

- **控制芯片**：
  - 74HC595D - 8位串行输出，控制电机和LED

### 物理布局

```
                   进纸方向 →
┌────────────────────────────────────────────────────────────┐
│                                                             │
│   进纸器电机     -------- 3cm --------  纸张传感器         │
│   (Feed Motor)              (Sensor)                       │
│   (GPIO32-Q7)              (GPIO34)                        │
│                                                             │
│                           3cm                               │
│                           ↓                                 │
│                                                             │
│                   面板电机   拾落电机                       │
│                  (Panel)   (Clamp)                         │
│                  (Q4-Q5)   (Q2-Q3)                         │
│                                                             │
└────────────────────────────────────────────────────────────┘
                          ↑
                        出纸方向
```

**关键距离**:
- **进纸器电机 → 传感器**: 3cm（30mm，2400步）
- **传感器 → 面板电机**: 3cm（30mm，2400步）
- **拾落电机（夹紧）**: 位于面板附近

## 🔄 状态机设计

### 状态定义

```cpp
typedef enum {
    PAPER_IDLE,              // 空闲状态 - 系统待机
    PAPER_PRE_CHECK,         // 预检状态 - 检测面板是否有纸
    PAPER_EJECTING,          // 出纸状态 - 送出已写完的纸张
    PAPER_FEEDING,           // 进纸状态 - 送入新纸张
    PAPER_UNCLAMP_FEED,      // 松开进纸状态 - 松开夹具并送纸5cm
    PAPER_CLAMP_FEED,        // 夹紧进纸状态 - 夹紧纸张
    PAPER_FULL_FEED,         // 完整进给状态 - A4行程进给
    PAPER_REPOSITION,        // 重新定位状态 - 精确定位到写字位置
    PAPER_COMPLETE,          // 完成状态 - 换纸完成
    PAPER_ERROR              // 错误状态 - 系统异常
} paper_change_state_t;
```

## 🔄 完整换纸流程

### 场景 1: 面板上有纸（正常换纸流程）

```
┌──────────────────────────────────────────────────────────┐
│ 步骤1: PRE_CHECK（预检）                                  │
├──────────────────────────────────────────────────────────┤
│ • 面板电机反转 2.5mm（200步）                             │
│ • 清除传感器前的遮挡                                       │
│ • 检测传感器状态                                          │
│ • 检测到纸张 → 进入 EJECTING                              │
└──────────────────────────────────────────────────────────┘
                           ↓
┌──────────────────────────────────────────────────────────┐
│ 步骤2: EJECTING（出纸）- 步骤1：边缘检测                  │
├──────────────────────────────────────────────────────────┤
│ • 面板电机反转，寻找纸张后边缘                            │
│ • 最大反转距离: 50mm（4000步）                            │
│ • 检测传感器跳变: 无纸(false) → 有纸(true)                │
│ • 表示检测到纸张后边缘                                    │
│ • 检测到后边缘: 重置step_counter=0，进入步骤2             │
└──────────────────────────────────────────────────────────┘
                           ↓
┌──────────────────────────────────────────────────────────┐
│ 步骤3: EJECTING（出纸）- 步骤2：完全出纸                  │
├──────────────────────────────────────────────────────────┤
│ • 面板电机正转，快速出纸（2kHz）                          │
│ • 送出距离: A4长度(297mm) + 预留(40mm) = 337mm            │
│ • 总步数: 23760 + 3200 = 26960 步                        │
│ • 完成后 → 进入 FEEDING                                   │
└──────────────────────────────────────────────────────────┘
                           ↓
┌──────────────────────────────────────────────────────────┐
│ 步骤4: FEEDING（进纸）                                    │
├──────────────────────────────────────────────────────────┤
│ • 进纸电机正转（只正转，永不反转）                        │
│ • 等待传感器检测到纸张前边缘                              │
│ • 传感器跳变: 无纸(false) → 有纸(true)                    │
│ • 检测到前边缘: 立即停止所有电机                          │
│ • 进入 UNCLAMP_FEED                                       │
│ • 超时保护: 500步（约6.25mm）内未检测到纸张 → 报错       │
└──────────────────────────────────────────────────────────┘
                           ↓
┌──────────────────────────────────────────────────────────┐
│ 步骤5: UNCLAMP_FEED（松开进纸）                           │
├──────────────────────────────────────────────────────────┤
│ • 阶段1: 夹紧电机正转 1.5cm（1200步），松开夹具           │
│ • 阶段2: 面板电机正转 3.5cm（2800步），纸张通过传感器     │
│ • 总步数: 4000 步                                         │
│ • 完成后 → 进入 CLAMP_FEED                                │
└──────────────────────────────────────────────────────────┘
                           ↓
┌──────────────────────────────────────────────────────────┐
│ 步骤6: CLAMP_FEED（夹紧进纸）                             │
├──────────────────────────────────────────────────────────┤
│ • 夹紧电机反转 1.5cm（1200步），回到初始位置              │
│ • 纸张被夹紧                                              │
│ • 完成后 → 进入 FULL_FEED                                 │
└──────────────────────────────────────────────────────────┘
                           ↓
┌──────────────────────────────────────────────────────────┐
│ 步骤7: FULL_FEED（完整进给）                              │
├──────────────────────────────────────────────────────────┤
│ • 进纸电机 + 面板电机同步运行（1kHz）                     │
│ • 进给 A4 长度（297mm，23760步）                          │
│ • 完成条件:                                                │
│   - 已进给A4长度 且 纸张脱离传感器（正常情况）            │
│   - 或 纸张提前脱离传感器（至少进给80%）                  │
│ • 完成后 → 进入 REPOSITION                                │
└──────────────────────────────────────────────────────────┘
                           ↓
┌──────────────────────────────────────────────────────────┐
│ 步骤8: REPOSITION（重新定位）                             │
├──────────────────────────────────────────────────────────┤
│ • 进纸电机停止                                            │
│ • 阶段1: 面板电机反转，寻找纸张前边缘（最多50mm）         │
│   - 检测传感器跳变: 无纸(false) → 有纸(true)              │
│   - 检测到前边缘: 重置step_counter=0                      │
│ • 阶段2: 面板电机正转 3cm（2400步），精确定位             │
│ • 完成后 → 进入 COMPLETE                                  │
└──────────────────────────────────────────────────────────┘
                           ↓
┌──────────────────────────────────────────────────────────┐
│ 步骤9: COMPLETE（完成）                                   │
├──────────────────────────────────────────────────────────┤
│ • 换纸完成，记录日志                                      │
│ • 调用 m0_paper_change_complete() 通知系统                │
│ • 返回 IDLE 状态                                          │
└──────────────────────────────────────────────────────────┘
```

## 📝 各状态详细说明

### 1. PAPER_IDLE（空闲状态）

**功能**: 系统待机，等待触发

**触发条件**:
- 用户按下换纸按钮（GPIO35，长按2秒=紧急停止）
- G-code 命令 `M0`（程序暂停）
- 一键换纸命令

**进入条件**: 
- 系统初始化完成
- 上一流程完成
- 错误恢复后

**代码位置**: `PaperChangeLogic.cpp:227-230`

---

### 2. PAPER_PRE_CHECK（预检状态）

**功能**: 快速检查面板上是否有纸张，决定流程分支

**执行流程**:
```
1. 面板电机反转 2.5mm（200步，约0.2秒）
   - 目的: 清除传感器前可能的遮挡
   
2. 检测传感器状态
   ├─ 检测到纸张 → 进入 EJECTING（需要先出纸）
   └─ 未检测到纸张 → 进入 FEEDING（直接进纸）
```

**关键参数**:
- `PAPER_PRE_CHECK_STEPS` = 200 步（2.5mm，80步/mm）
- 执行时间: 约 0.2 秒

**安全保护**:
- 最大步数: `PAPER_PRE_CHECK_STEPS + 50`
- 状态超时: `PAPER_STATE_TIMEOUT_MS` (30秒)

**代码位置**: `PaperChangeLogic.cpp:293-320`

**优化说明**:
- ✅ 这是唯一的预检步骤（优化后移除了EJECTING的重复预检）
- ✅ 距离短（2.5mm），速度快，快速决策

---

### 3. PAPER_EJECTING（出纸状态）

**功能**: 将面板上的纸张完全送出

**优化后流程（2个步骤）**:

#### 步骤 1: 边缘检测（寻找纸张后边缘）

**物理流程**:
```
1. 纸张在面板上（传感器前方，约3cm外）
   ↓
2. 面板电机反转，纸张向后移动
   ↓
3. 纸张后边缘经过传感器位置
   ↓
4. 传感器跳变: 无纸(false) → 有纸(true)
   ↓
5. 检测到后边缘，重置计数器，准备正转出纸
```

**代码流程**:
```cpp
• 从 step_counter = 0 开始（由 update_control_structure() 重置）
• 面板电机反转，寻找纸张后边缘
• 最大反转距离: PAPER_MAX_REVERSE_STEPS = 4000步（50mm）
• 检测传感器跳变: 无纸(false) → 有纸(true)
  - 表示检测到纸张后边缘
  - 传感器位置: 距离面板电机3cm，距离进纸器电机3cm
  - 反转时: 纸张后边缘经过传感器
• 检测到后边缘: 重置 step_counter=0，设置 eject_detected=true
• 设置快速正转速度（2kHz，500μs间隔）
```

**关键参数**:
- `PAPER_MAX_REVERSE_STEPS` = 4000 步（50mm）
- 检测速度: 正常速度（1000μs间隔）

#### 步骤 2: 完全出纸

```cpp
• 从 step_counter = 0 开始（检测到边缘后重置）
• 面板电机正转，快速出纸（2kHz，500μs间隔）
• 送出距离: A4长度(297mm) + 预留(40mm) = 337mm
• 总步数: PAPER_EJECT_STEPS + PAPER_3TO5CM_STEPS = 23760 + 3200 = 26960步
• 完成后: eject_detected=false，进入 FEEDING
```

**关键参数**:
- `PAPER_EJECT_STEPS` = 23760 步（297mm，A4长度）
- `PAPER_3TO5CM_STEPS` = 3200 步（40mm预留）
- 快速出纸速度: 2kHz（500μs间隔）

**代码位置**: `PaperChangeLogic.cpp:328-365`

**优化说明**:
- ✅ **移除了步骤0的预检**（原来反转3cm，现已优化）
- ✅ PRE_CHECK已经检测到有纸，无需再次预检
- ✅ 直接从边缘检测开始，节省时间（约2.4秒）

### 4. PAPER_FEEDING（进纸状态）

**功能**: 从纸盒送入新纸张

**物理流程**:
```
1. 进纸器电机正转（从纸盒送入纸张）
   ↓
2. 纸张向前移动，经过进纸器电机位置
   ↓
3. 继续前进 3cm（2400步），到达传感器位置
   ↓
4. 传感器检测到纸张前边缘（跳变: 无纸→有纸）
   ↓
5. 停止进纸电机，精确定位
```

**执行流程**:
```cpp
1. 进纸电机正转（注意：进纸电机只正转，永不反转）
   - 步进间隔: PAPER_FEED_INTERVAL_US = 2000μs（低速进纸）
   
2. 监控传感器，等待检测到纸张前边缘
   - 传感器跳变: 无纸(false) → 有纸(true)
   - 表示检测到纸张前边缘
   - 物理距离: 传感器距离进纸器3cm（2400步）
   
3. 检测到前边缘
   - 立即停止所有电机（精确定位）
   - 进入 UNCLAMP_FEED 状态
   
4. 超时保护
   - 500步（约6.25mm）内未检测到纸张 → 报错
   - 通常表示纸张盒为空或传感器故障
   - ⚠️ 注意：如果纸张从进纸器电机位置开始，需要2400步才能到传感器
```

**关键参数**:
- 超时保护: 500 步（约 6.25mm）
- 步进间隔: 2000μs（低速进纸）
- 最大步数: 1000 步
- **物理距离**: 进纸器电机 → 传感器 = 3cm（2400步）

**安全保护**:
- 最大步数: 1000 步
- 超时检测: 500 步内必须检测到纸张

**代码位置**: `PaperChangeLogic.cpp:473-495`

**要点**:
- ⚠️ **进纸电机只正转**，这是机械设计决定的
- ✅ 传感器检测到前边缘后立即停止，精确定位
- ⚠️ **注意**: 如果纸张从进纸器电机位置开始，需要移动3cm（2400步）才能到达传感器
- ⚠️ **超时保护**: 500步可能太小，如果纸张从进纸器开始，需要至少2400步

**物理位置说明**:
- 传感器位于进纸器电机和面板电机中间
- 距离进纸器电机3cm（2400步）
- 距离面板电机3cm（2400步）

### 5. PAPER_UNCLAMP_FEED（松开进纸状态）

**功能**: 松开夹具并送纸 5cm

**执行流程**:

#### 阶段 1: 松开夹具（1.5cm）

```cpp
• 夹紧电机正转 1.5cm（1200步）
• 松开夹具（初始状态为夹紧态）
• 进度报告: 每50步报告一次
```

#### 阶段 2: 面板送纸（3.5cm）

**物理流程**:
```
1. 面板电机正转，纸张向前移动
   ↓
2. 纸张经过传感器位置（距离面板电机3cm）
   ↓
3. 继续前进0.5cm（总共3.5cm）
```

**代码流程**:
```cpp
• 面板电机正转 3.5cm（2800步）
• 纸张经过传感器后继续运行
• 物理意义: 传感器到面板电机距离3cm，正转3.5cm使纸张经过传感器并继续前进
• 总步数: PAPER_1_5CM_STEPS + PAPER_3_5CM_STEPS = 4000步
```

**关键参数**:
- `PAPER_1_5CM_STEPS` = 1200 步（15mm）
- `PAPER_3_5CM_STEPS` = 2800 步（35mm）
- 总步数: 4000 步

**代码位置**: `PaperChangeLogic.cpp:501-544`

### 7. PAPER_FULL_FEED（完整进给状态）

**功能**: 两个电机同步运行，将纸张完全送入系统

**执行流程**:
```cpp
1. 进纸电机和面板电机同步运行
   - 步进间隔: 1000μs（1kHz）
   - 两个电机同步步进（相同时间戳）
   
2. 进给 A4 长度（297mm，23760步）
   
3. 检测完成条件:
   - 已进给A4长度 且 纸张脱离传感器（正常情况）
   - 或 纸张提前脱离传感器（至少进给80%，19008步）
   
4. 停止所有电机，进入 REPOSITION
```

**同步控制逻辑**:
```cpp
// 只有当两个电机都到了应该步进的时间时，才同步执行步进
if (millis() - feed_timing->last_step_time >= feed_timing->step_interval &&
    millis() - panel_timing->last_step_time >= panel_timing->step_interval) {
    
    // 同步步进两个电机
    motor_step_control(BIT_FEED_MOTOR_STEP, ...);
    motor_step_control(BIT_PANEL_MOTOR_STEP, ...);
    
    // 更新时间戳（相同时间）
    feed_timing->last_step_time = millis();
    panel_timing->last_step_time = millis();
    ctrl->step_counter++;
}
```

**关键参数**:
- `PAPER_EJECT_STEPS` = 23760 步（297mm）
- 最小进给: 80% × 23760 = 19008 步
- 同步速度: 1kHz（1000μs间隔）

**代码位置**: `PaperChangeLogic.cpp:586-628`

### 8. PAPER_REPOSITION（重新定位状态）

**功能**: 精确定位纸张到写字位置

**执行流程**:

#### 阶段 1: 反转寻找边缘（最多50mm）

**物理流程**:
```
1. 进纸电机停止
   ↓
2. 面板电机反转，纸张向后移动
   ↓
3. 纸张前边缘经过传感器位置（距离面板电机3cm）
   ↓
4. 传感器跳变: 无纸(false) → 有纸(true)
   ↓
5. 检测到前边缘，重置计数器
```

**代码流程**:
```cpp
• 进纸电机停止
• 面板电机反转，寻找纸张前边缘
• 最大反转: PAPER_MAX_REVERSE_STEPS = 4000步（50mm）
• 检测传感器跳变: 无纸(false) → 有纸(true)
  - 表示检测到纸张前边缘（反转时检测前边缘）
  - 传感器距离面板电机3cm，反转寻找边缘
• 检测到前边缘: 重置 step_counter=0，设置 reverse_complete=true
```

#### 阶段 2: 正转定位（3cm）

**物理流程**:
```
1. 检测到前边缘后（传感器位置）
   ↓
2. 面板电机正转3cm（2400步）
   ↓
3. 将纸张精确定位到写字位置
```

**代码流程**:
```cpp
• 检测到前边缘后，从 step_counter=0 开始
• 面板电机正转 3cm（2400步）
• 关键步骤！确保纸张位置正确
• 物理意义: 从传感器位置正转3cm，将纸张定位到写字位置
• 完成后: reverse_complete=false，进入 COMPLETE
```

**注意**: 3cm定位距离可能对应：
- 传感器到写字位置的距离，或
- 传感器到面板电机的距离（3cm）

**关键参数**:
- `PAPER_MAX_REVERSE_STEPS` = 4000 步（50mm）
- `PAPER_3CM_STEPS` = 2400 步（30mm）

**代码位置**: `PaperChangeLogic.cpp:634-692`

**要点**:
- ✅ 这是关键步骤，确保纸张位置正确
- ✅ 反转找到边缘，然后正转3cm精确定位

- **控制位**: Q2 (DIR), Q3 (STEP)

- **方向**:
  - 正转: 松开夹具
  - 反转: 夹紧纸张（回到初始位置）

## 🔍 传感器检测机制

### 传感器特性

- **引脚**: GPIO34（只能输入，无法上拉）
- **逻辑**: 低电平有效
  - `LOW` (0V) = 检测到纸张 → `paper_sensor_state = true`
  - `HIGH` (3.3V) = 无纸张 → `paper_sensor_state = false`

### 去抖动处理

```cpp
bool read_paper_sensor_debounced() {
    static bool last_state = false;
    static uint32_t last_change_time = 0;
    static bool stable_state = false;
    
    bool current_state = read_paper_sensor_raw();
    uint32_t current_time = millis();
    
    if (current_state != last_state) {
        last_change_time = current_time;
        last_state = current_state;
    } else if (current_time - last_change_time > PAPER_SENSOR_DEBOUNCE_MS) {
        stable_state = current_state;
    }
    
    return stable_state;
}
```

**去抖动时间**: `PAPER_SENSOR_DEBOUNCE_MS` = 50ms

### 边缘检测

**后边缘检测**（反转时）:
```cpp
// 传感器从无纸(false)变为有纸(true)
if (ctrl->paper_sensor_state && !ctrl->last_paper_sensor_state) {
    // 检测到纸张后边缘
}
```

**前边缘检测**（正转时）:
```cpp
// 传感器从无纸(false)变为有纸(true)
if (ctrl->paper_sensor_state && !ctrl->last_paper_sensor_state) {
    // 检测到纸张前边缘
}
```

**要点**:
- ✅ 通过状态跳变检测边缘
- ✅ 需要保存上一状态 (`last_paper_sensor_state`)
- ✅ 在主循环中更新传感器状态

## 🛡️ 安全保护机制

### 1. 状态超时保护

```cpp
bool runtime_safety_check(uint32_t step_counter, uint32_t max_steps, const char* operation_name) {
    // 步数溢出检查
    if (step_counter >= max_steps) {
        LOG_ERROR_F("Safety limit exceeded...");
        STOP_ALL_MOTORS();
        enter_state(PAPER_ERROR);
        return false;
    }
    
    // 状态时间检查（安全处理millis()溢出）
    uint32_t current_time = millis();
    uint32_t state_duration;
    if (current_time >= ctrl->state_timer) {
        state_duration = current_time - ctrl->state_timer;
    } else {
        // 处理millis()溢出情况（49天后）
        state_duration = (UINT32_MAX - ctrl->state_timer) + current_time + 1;
    }
    
    if (state_duration > PAPER_STATE_TIMEOUT_MS) {
        LOG_ERROR_F("State timeout...");
        STOP_ALL_MOTORS();
        enter_state(PAPER_ERROR);
        return false;
    }
    
    return true;
}
```

**超时参数**: `PAPER_STATE_TIMEOUT_MS` = 30000ms (30秒)

### 2. 状态转换验证

```cpp
bool is_valid_transition(paper_change_state_t from_state, paper_change_state_t to_state) {

        to_state < PAPER_IDLE || to_state > PAPER_ERROR) {
```

```
  ↓
EJECTING 步骤2: 正转出纸
```
